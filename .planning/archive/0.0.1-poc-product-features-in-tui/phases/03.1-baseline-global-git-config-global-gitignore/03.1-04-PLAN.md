`setup` (backup→preview→confirm→write, `--dry-run`) and `show` (sidecar-free read-back),
registered on the root command. After this plan, a real user can run `gitid baseline

them into the user-facing command following the exact Phase 2/3 thin-Cobra-handler
pattern (`add.go`/`list.go`). All business logic stays in `internal/`; `cmd/` only
gathers input, runs the preview/confirm flow, and prints.

group added to `cmd/gitid/main.go`.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| terminal stdin → runBaselineSetup prompts | User keystrokes select Tier-2/rewrites and confirm; routed through the existing `prompt`/`confirm` helpers, not interpolated into commands |
| cmd → internal/gitconfig writers | All mutation delegated to the validated, tested internal layer; cmd performs no direct file I/O |
| os.UserHomeDir() → file paths | Paths derived in-process; the literal `~/.gitconfig.d/00-baseline` written into the include is a fixed constant, not user input (T-03.1-03 carried) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03.1-08 | Spoofing/Repudiation | confirm gate before write | mitigate | Single explicit `confirm(... "Write baseline now?")` (default N) before any write; `--dry-run` short-circuits earlier. No write path proceeds without confirmation (SAFE-03). |
| T-03.1-09 | Tampering (V5) | Tier-2/rewrite user edits → internal renderers | mitigate | User-supplied strings flow into the plan-02 renderers, which call `validateValue` (newline/`[remote` guard) before render — the cmd layer adds no unvalidated render path. |
| T-03.1-10 | Info Disclosure | backups of `~/.gitconfig` / `00-baseline` | mitigate | Writers route through `filewriter.Write` (backups 0o600); cmd never writes files directly (`grep` gate asserts no `os.WriteFile`). |
| T-03.1-SC | Tampering | npm/pip/cargo installs | accept | No package installs (no new dependencies — cobra already vendored). No legitimacy gate needed. |
</threat_model>
