<objective>
Add the `PrependBlockIfNotFound` pure function to `internal/filewriter` — the shared
foundation primitive every Phase 03.1 slice depends on. It places a managed block at
the TOP of a file on first write (so git evaluates the `[include]` as a FLOOR, per
D-10 / RESEARCH C1), and delegates to the existing `ReplaceBlock` for in-place update
on re-runs (preserving the floor position).

Purpose: `ReplaceBlock` appends on first write, which would make the baseline
`[include]` a CEILING that silently overrides user keys (RESEARCH Pitfall A). The
floor model requires a prepend variant. This is the one new low-level primitive the
MVP needs; all higher slices compose it.
Output: `PrependBlockIfNotFound` in `internal/filewriter/block.go` (additive, no new
file) + RED/GREEN tests in `internal/filewriter/block_prepend_test.go`.
</objective>
