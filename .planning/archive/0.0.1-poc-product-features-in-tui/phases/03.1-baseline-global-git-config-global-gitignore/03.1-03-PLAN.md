<objective>
Add the read-side of the baseline to `internal/gitconfig/baseline.go`: `ScanConflicts`
(D-11 — detect keys the user already set that the baseline also sets, so the preview
can show current/baseline/winner under the floor model) and `ReadBaselineState` (SC-5 —
re-derive the managed baseline from disk with no sidecar DB), plus the independent
removal of the `url-rewrites` block (URLRW-01 deselect-all — the `url-rewrites` block is independently removable per D-07).

Purpose: The conflict scan cannot use `git config --file --list` directly — it would
report managed-block keys as user keys (RESEARCH C2 / Pitfall C). The correct algorithm
strips all managed blocks first, then parses the user-owned remainder. The read-back
mirrors the IDENT-07 reconstruction model (`ListBlocks` across files), keeping gitid
sidecar-free.
Output: `ScanConflicts`, `ReadBaselineState`, `RemoveURLRewritesBlock` (or equivalent)
in `internal/gitconfig/baseline.go` + TDD tests.
</objective>
