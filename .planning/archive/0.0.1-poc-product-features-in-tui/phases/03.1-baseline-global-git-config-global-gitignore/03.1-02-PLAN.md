<objective>
Build the baseline render + write layer in a new `internal/gitconfig/baseline.go`:
byte-stable renderers for the baseline toggle block, the url-rewrites block, and the
gitignore block; plus the writers that compose them through the `filewriter`
chokepoint. This is the domain-logic slice that makes the three managed surfaces real
and idempotent (SC-1/SC-2). It composes `PrependBlockIfNotFound` (plan 01) for floor
placement of the `[include]` block. The baseline lives in `~/.gitconfig.d/00-baseline` wired via a managed `[include]` block (D-01).

Purpose: GLOBAL-01 (baseline toggles + aliases), URLRW-01 (insteadOf rewrites), and
GITIGNORE-01 (managed `~/.gitignore_global` + `core.excludesfile`) all reduce to
deterministic raw-text renders written via `ReplaceBlock`/`Write` — never via `git
config set` on a sentinel-bearing file (RESEARCH C3 / Pitfall B). Idempotency is the
contract: write → re-write → byte-identical. The baseline file is owned by a managed block inside it; foreign content is preserved verbatim (D-02).
Output: `internal/gitconfig/baseline.go` with the renderers and writers below, plus
TDD tests in `internal/gitconfig/baseline_test.go`.
</objective>

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| user input (URL rewrite values, optional pager/alias overrides) → renderers | Untrusted strings could carry `\n` / `[remote` injection into a written gitconfig file |
| in-process constants → `include.path` body | The include path is a fixed in-process string derived in plan 04 from `os.UserHomeDir()`, never user input |
| baseline.go → filewriter.Write | The single safe-write chokepoint (backup + atomic + chmod) — no bypass |
