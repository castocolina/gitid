<decisions>
## Implementation Decisions

### Create-flow loop (FIX-CREATE-01 / G-05, F-2)
- **D-01:** Generate the **real key pair into `~/.ssh` immediately** (so the `.pub`
  is on disk, uploadable, and reusable on rerun). Only the **four config artifacts**
  (SSH Host block, includeIf, fragment, allowed_signers) wait for an authenticated
  PASS. This replaces the current temp-stage→delete behavior (`add.go:337`/`Cleanup`)
  that caused the "key vanished / inaccessible on rerun" complaint.
- **D-02:** New ordering: generate key → copy `.pub` + show upload instructions →
  **wait** → loop the connectivity test on demand → on authenticated PASS, persist.
  The old "Write all four artifacts now?" prompt **before** any test (`add.go:312`)
  is removed.
- **D-03:** On **authenticated PASS → auto-persist** the four artifacts and report
  backup paths. The PASS result IS the gate — no extra confirm prompt.
- **D-04:** The loop offers retry / skip-&-write / quit. **Quit before PASS keeps the
  generated key** in `~/.ssh`, prints its location, and notes that `doctor` will flag
  it as an orphan until config is written. No config artifacts are written on quit.
- **D-05:** The **`[s]` skip-&-write escape** (offline / upload-later) writes all four
  artifacts without an authenticated PASS, but **requires an explicit typed confirm**
  with a "not yet authenticated; you must upload the key later" warning first.
- **D-06:** PASS = **authenticated** success (e.g. GitHub's "successfully
  authenticated" banner), NOT mere reachability. The current bug treats
  `Permission denied (publickey)` ("ReachableNotUploaded") as good-enough — that must
  no longer satisfy the gate.

### Match strategy (MATCH-URL-01 / F-5, F-6)
- **D-07:** `identity add` prompts a **picker first** — `1) folder (gitdir)`,
  `2) repo URL (hasconfig)`, `3) both` — then prompts only for the value(s) the choice
  needs. Folder (`gitdir`) stays the default. "both" writes two `[includeIf]`
  conditions (git applies them additively / OR; `RenderIncludeIf` already takes a
  `[]Match`).
- **D-08:** The URL pattern prompt **suggests a default** derived from the host
  alias's hostname + identity name, e.g. `git@ssh.github.com:<name>/**` (user edits).
  Value is written as `hasconfig:remote.*.url:<pattern>`.
- **D-09:** Wire match-strategy editing into **`identity update`** too (change/add
  gitdir/URL/both on an existing identity), and add **non-interactive flags**
  (`--name/--gitdir/--url/--provider`) to `add`/`update`. *(Deliberate scope addition
  beyond the ROADMAP success criteria — confirmed by the user; planner should add the
  matching requirement(s) and note the expanded surface.)*
- **D-10:** Relabel the `alias` prompt → **"Host alias"** (F-5).

### Provider reconstruction (FIX-RECON-01 / F-3)
- **D-11:** Persist the user-entered provider as a **marker comment inside the SSH
  managed Host block** (e.g. `# gitid: provider=github`), invisible to `ssh` itself.
  The `sshconfig` parser (`ParseManagedHosts`) reads it back.
- **D-12:** On read, **prefer the marker; fall back to a hostname→provider map**
  (`ssh.github.com`/`github.com`→`github`, `altssh.gitlab.com`/`gitlab.com`→`gitlab`)
  for legacy/markerless blocks and custom hosts. Replaces the brittle
  `TrimPrefix(alias, name+".")` derivation at `loader.go:46-51`.
- **D-13:** **Legacy identities** (no marker, e.g. current `castocolina`,
  `user_z3r0_gh`) are **tolerated via the fallback — no forced rewrite/migration**.
  Markers get added naturally on the next `update`/`rotate`.

### Doctor overlap check (DOC-08 / F-7)
- **D-14:** Overlap detection flags **all three** kinds: identical `gitdir`,
  nested/prefix `gitdir` (parent vs child dir), and overlapping `hasconfig` URL
  patterns (conservative heuristic acceptable for URL globs).
- **D-15:** Severity = **warning** (config still resolves last-wins; report which
  identity currently wins + how to disambiguate). Exit code stays in the warning tier.
- **D-16:** Detection is a **shared function** consumed by both `doctor` and the
  `add`/`update` write paths. At write time, an overlap triggers **warn + explicit
  y/N to proceed** (deliberate overlaps stay possible; pairs with auto-persist-on-PASS).

### Install feedback (FIX-INSTALL-01 / F-1)
- **D-17:** `gitid` **self-reports** the binary path + whether it is on `$PATH`
  (surfaced via doctor and/or an install/where hint), so the hint also works for a
  direct `go install` that bypasses `make`. `make install` also prints the resolved
  install path + a PATH export hint when not on `$PATH`.

### Testing & verification strategy (added 2026-06-14)
- **D-18:** Plans use **three test layers**: (1) **TDD unit tests** (per CLAUDE.md,
  RED→GREEN→REFACTOR) for all eligible logic; (2) **deterministic hermetic E2E** —
  exec the built binary under a sandbox `HOME` with a **fake `ssh` injected on PATH**
  (emits the GitHub/GitLab "successfully authenticated" banner vs
  `Permission denied (publickey)`) so the auth-PASS gate + retry/skip/quit loop are
  CI-runnable and regression-proof; (3) **agent-driven experience evaluation** (D-19).
- **D-19:** An **agent-driven experience-evaluation** pass: an agent drives the
  **real built CLI** through the flows this phase changes — create-flow loop,
  `identity add`/`update` match picker, `identity list` provider, `doctor` overlap,
  install feedback — and produces a **structured UX/experience findings report**
  (what works, what's confusing, what to fix). **Scope = only the CLI flows 5.5
  fixes** (NOT the TUI). **Advisory, not a gate** — captured and feeds Phase 5.6;
  the deterministic tests (D-18 layers 1-2) are the actual completion gate.
- **D-20:** The fake-ssh injection seam is **PATH prepending** (the runner resolves
  `ssh` via PATH, confirmed in `internal/tester`); the fake `ssh` is a static script
  written via `os.WriteFile` at `0o700` — **never** `exec.Command("sh","-c",...)`
  (avoids gosec G204). Any new seam to inject the ssh binary stays test-only.

### Claude's Discretion
- Exact wording of prompts/menus, the precise PASS-detection substring set (reuse
  `internal/tester` classifier), and the URL-overlap heuristic implementation are
  left to research/planning, within the decisions above.
- Researcher open questions (planner to resolve): (Q1) `--dry-run` should skip keygen
  entirely; (Q2) `update` keeps its positional name arg (`--name` flag only on `add`);
  (Q3) `update --provider` overwrites the marker via flag only, no interactive prompt.

<code_context>
## Existing Code Insights

### Reusable Assets
- `internal/gitconfig` `Match`/`RenderIncludeIf`/condition parser: both `gitdir` and
  `hasconfig` are already modeled — MATCH-URL needs only prompt + flag plumbing, not
  new rendering.
- `internal/tester` two-phase classifier (`PreWrite`, `Resolved`): reuse for the
  loop's authenticated-PASS gate; do NOT re-implement reachability classification.
- `identity.Deps` seam (`add.go:319` `buildDeps`): the loop re-orders calls to the
  existing `Generate`/`PersistKey`/`WriteSSH`/`WriteGitconfig`/... seam; persistence
  boundary moves, the writers don't.
- `filewriter` chokepoint (backup + atomic + idempotent managed block): all writes,
  including the marker comment, must route through it.

### Established Patterns
- No-sidecar reconstruction (IDENT-07): provider marker MUST live inside a managed
  block (D-11), never a separate state file.
- Reserved-block carve-out (`baseline-include`, `_global`): the overlap check must
  exclude reserved/global blocks the same way doctor's identity/orphan checks do.
- Shared doctor detection feeding write-time warnings mirrors the existing
  doctor-as-library pattern.

### Integration Points
- `add`/`update` call the shared overlap detector before persisting (D-16).
- `Reconstruct` consumes the SSH marker + hostname-map fallback for provider (D-12).
- `make install` + `gitid` doctor/where both surface the install-path/PATH hint (D-17).

</code_context>

<specifics>
## Specific Ideas

- Provider marker format: `# gitid: provider=<provider>` inside the managed SSH Host
  block (D-11).
- URL default shape: `git@<hostname>:<name>/**` → written as
  `hasconfig:remote.*.url:git@<hostname>:<name>/**` (D-08).
- "both" = two `[includeIf]` conditions, OR-applied by git (D-07).
- Authenticated-PASS, not reachability, is the persist gate (D-06) — the central
  reconciliation fix.

</specifics>
