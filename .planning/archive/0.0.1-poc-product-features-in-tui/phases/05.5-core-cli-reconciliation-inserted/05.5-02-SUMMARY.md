# Phase 05.5 Plan 02: Provider Marker Render + Parse Summary

**One-liner:** Provider persisted as `# gitid: provider=<p>` in SSH Host block, parsed back via `extractProviderMarker` pre-pass, reconstructed via marker-prefer then hostname-map fallback; F-3 TrimPrefix bug removed.

## What Was Built

### Task 1: Provider marker — render + parse round-trip (RED→GREEN)

**RED (ee3948c):** Added 6 new failing tests + stubs with `_`-param convention (no `--no-verify`):
- `TestRenderHostBlock_WithProvider`: marker emitted after `IdentitiesOnly yes` (D-11)
- `TestRenderHostBlock_EmptyProvider`: no marker when provider is empty (legacy safe)
- `TestRenderHostBlock_NewlineInProviderPanics`: security guard (T-05.5-04)

**GREEN (0052d93):** Real implementations replacing stubs:

- `SSHHostInfo.Provider string` field added to struct
