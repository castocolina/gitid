## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 1 - Bug] Missing `~/.ssh` directory creation before key generation**
- **Found during:** Task 1 (first run of create E2E tests)
- **Issue:** `Generate` dep in `buildDeps` called `filewriter.Write(finalPriv, ...)` without first ensuring `~/.ssh` exists. `filewriter.Write` uses a temp file in the target directory, which fails with `no such file or directory` when `~/.ssh` has never been created (e.g. a fresh sandbox HOME in E2E tests).
- **Fix:** Added `filewriter.EnsureDir(sshDir, 0o700)` call before `keygen.KeyPaths` in the `Generate` closure in `cmd/gitid/add.go`. This is idempotent — existing installs are unaffected.
- **Files modified:** `cmd/gitid/add.go`
- **Commit:** 63ede03

**2. [Rule 1 - Bug] Wrong command path in overlap and install E2E tests**
- **Found during:** Task 1 (reading overlap/install test output: doctor subcommand help shown instead of doctor report)
- **Issue:** `overlap_e2e_test.go` and `install_e2e_test.go` invoked `bin, "identity", "doctor"` — but `doctor` is a ROOT-level command (`gitid doctor`), not a sub-command of `identity`. This caused Cobra to print the identity subcommand help and exit non-zero.
- **Fix:** Changed both files to use `bin, "doctor"`.
- **Files modified:** `e2e/overlap_e2e_test.go`, `e2e/install_e2e_test.go`
- **Commit:** 63ede03

**3. [Rule 1 - Bug] Fake-ssh stub missing `ssh -Q key` handler**
- **Found during:** Task 1 (all `identity add` E2E tests failed with `probing ssh key types via 'ssh -Q key': exit status 1`)
- **Issue:** `FakeSSHDir` static script handled `-G` (Resolved check) and connection modes but not `-Q` (key algorithm query). `ProbeKeyTypes()` in `platform/platform.go` calls `ssh -Q key` before any prompt, so ALL create/match tests failed immediately.
- **Fix:** Added `-Q)` branch (before `-G`) emitting three static key-type tokens (`ssh-ed25519`, `ssh-rsa`, `ecdsa-sha2-nistp256`) and exiting 0.
- **Files modified:** `e2e/harness_test.go`
- **Commit:** 63ede03

**4. [Rule 1 - Bug] Stdin sequences in create tests not matching implemented prompt order**
- **Found during:** Task 1 (QuitKeepsKey and SkipRequiresConfirm failed: `"q"` and `"s"` were consumed as passphrase instead of the loop prompt)
- **Issue:** The 05.5-01 RED shells approximated the match-strategy prompt with a single `""` for gitdir. After 05.5-04 implemented `promptMatchStrategy`, TWO additional stdin lines are now consumed: the strategy choice (default "1") and the gitdir value. The `"q"` and `"s"` loop responses ended up at the passphrase prompt, not the loop prompt.
- **Fix:** Updated `TestCreateFlow_QuitKeepsKey` and `TestCreateFlow_SkipRequiresConfirm` to insert `""` for strategy and `""` for gitdir before passphrase and loop inputs.
- **Files modified:** `e2e/create_e2e_test.go`
- **Commit:** 63ede03

**5. [Rule 2 - Missing functionality] TestListProvider_MarkerlessFallback test added**
- **Found during:** Task 1 planning (acceptance criteria required `identity list` showing `provider: github` for a markerless block)
- **Issue:** The 05.5-01 RED shells had no test covering FIX-RECON-01 end-to-end (`identity list` + hostname-to-provider fallback). The plan's acceptance criteria explicitly required it.
- **Fix:** Added `TestListProvider_MarkerlessFallback` to `e2e/match_e2e_test.go`: seeds a markerless SSH managed block with `Hostname ssh.github.com`, runs `gitid identity list`, asserts `provider: github` in output (D-12 fallback).
- **Files modified:** `e2e/match_e2e_test.go`
- **Commit:** 63ede03

**6. [Rule 3 - Blocking] `t.TempDir` module-cache cleanup permission errors**
- **Found during:** Task 1 (tests showed FAIL-only-from-cleanup despite assertion passing)
- **Issue:** `BuildBinary` used `t.TempDir()` for the binary dir. When `SandboxHome(t)` changes `HOME` before `BuildBinary` runs, `go build` uses `$HOME/go` as the default GOPATH, storing the module cache in the sandbox temp dir. The test framework's cleanup of `t.TempDir()` fails because `go.mod` files in the module cache are read-only, causing all tests using `BuildBinary` to FAIL even when assertions passed.
- **Fix:** Refactored `BuildBinary` to use `sync.Once` + `os.MkdirTemp` (not `t.TempDir`): binary is built once per package run into an unmanaged temp directory, and `HOME` is restored to `realHome` (captured at package init time, before any `t.Setenv`) so `go build` uses the real GOPATH.
- **Files modified:** `e2e/harness_test.go`
- **Commit:** 63ede03
