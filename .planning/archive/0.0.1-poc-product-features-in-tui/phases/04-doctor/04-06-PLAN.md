<objective>
Close DOC-GAP-01: `gitid doctor --fix` currently reports `fixed:` and increments
the applied tally for orphan / coherence / baseline findings while touching ZERO
files, because those findings carry `Fix.Fn: func() error { return nil }` stubs.
The real chokepoint closures (`deps.RemoveBlock`, `deps.AddWiring`) wired in
`buildDoctorDeps` have zero production call sites.

This plan plumbs the real fixers through the check-layer `Fix.Fn` fields so that
auto-fix actually mutates the on-disk config (through the filewriter chokepoint:
timestamped backup + atomic write + idempotent managed block), and fixes the two
quality defects coupled to the fix path (WR-02 hardcoded 0600 mode for the 0644
allowed_signers file; WR-01 first-hit false email mismatch).

Purpose: a `fixed:` line must mean a file was actually changed — this is a
data-integrity/trust defect, not cosmetic.
Output: real fix wiring in the three check families, a mode-aware RemoveBlock
closure, an all-candidate findSignerLine scan, and integration tests that drive
the REAL buildDoctorDeps wiring against a temp HOME and assert on-disk effects.

Honors D-01 (doctor core never imports filewriter — the fix runs through injected
`deps.RemoveBlock`/`deps.AddWiring` closures owned by the cmd layer), D-02
(auto-fix classes: orphan-block removal + missing-wiring re-add), and D-03
(report-only families stay Fix=nil).
</objective>
