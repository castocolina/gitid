## Artifacts this phase produces (this plan's contribution)

- `internal/keygen/keygen.go`
  - `func Generate(algo, identity, comment, passphrase string) (Result, error)` — ed25519 path (algo "ed25519"); returns `Result{ PrivatePath, PubPath, PubLine string }`
  - Key filename convention: `~/.ssh/id_<algo>_<identity>` (D-06), `.pub` alongside
  - Writes delegate to `filewriter.Write(path, bytes, 0600|0644)` — never `os.WriteFile`
- `internal/keygen/signers.go`
  - `func AllowedSignersLine(email, pubLine string) string` — `<email> namespaces="git" <pubLine-without-trailing-newline>\n` (SIGN-01)
  - `func WriteAllowedSigners(path, identity, line string) (backupPath string, err error)` — persists `line` into the `~/.ssh/allowed_signers` file as an idempotent per-identity managed block keyed by `identity` (sentinels `# BEGIN gitid managed: <identity>` / `# END gitid managed: <identity>`), composed via `filewriter.ReplaceBlock` then written via `filewriter.Write(path, composed, 0644)`. This is the FOURTH coordinated artifact; the file path matches the `gpg.ssh.allowedSignersFile` pointer 02-05 writes globally. Re-running with the same identity yields an empty diff; a different identity appends a distinct block without disturbing existing lines (SAFE-02). Never `os.WriteFile`.
- `internal/clipboard/clipboard.go`
  - `func Copy(text string) error` — wraps `atotto/clipboard.WriteAll`; returns a sentinel/typed error when no clipboard tool is available so the caller prints the key manually (CLIP-02)
- `internal/clipboard/doc.go` — phase marker updated from "Phase 5+" to Phase 2 (clipboard pulled forward per RESEARCH/ PATTERNS note)
- go.mod/go.sum: `golang.org/x/crypto@v0.53.0`, `github.com/atotto/clipboard@v0.1.4`

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| gitid → filesystem (`~/.ssh`) | newly generated private key + .pub + allowed_signers written here |
| gitid → OS clipboard | public key text placed on the system clipboard |
| gitid → third-party deps | x/crypto (key serialization) + atotto/clipboard |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-09 | Information Disclosure | private key written world-readable | mitigate | keygen writes the private key via `filewriter.Write(path, pem, 0600)`; .pub 0644; never os.WriteFile |
| T-02-10 | Cryptographic weakness | hand-rolled OpenSSH key serialization corrupts key | mitigate | use `ssh.MarshalPrivateKey`/`MarshalAuthorizedKey` from x/crypto (verified signatures); never hand-roll |
| T-02-11 | Spoofing | cross-protocol signing-key reuse / untrusted principal | mitigate | `AllowedSignersLine` mandates `namespaces="git"` and email byte-identical to user.email (SIGN-01) |
| T-02-33 | Tampering | duplicate / stale allowed_signers lines across re-runs or multiple identities | mitigate | `WriteAllowedSigners` uses an idempotent per-identity `ReplaceBlock` keyed by identity name; backup taken; foreign lines preserved (SAFE-02) |
| T-02-12 | Tampering | supply-chain (npm/pip/cargo-style) on new deps | mitigate | blocking-human legitimacy checkpoint above; go.sum checksums; Go modules run no install scripts (T-02-SC) |
| T-02-SC | Tampering | go get pulls a compromised module | mitigate | both deps pre-pinned + Approved in RESEARCH audit; checksums in go.sum verify integrity |
