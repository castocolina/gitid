<objective>
Build `internal/filewriter` — the single safe-write chokepoint every other writer in
Phase 2 delegates to. It implements timestamped backup, atomic temp→fsync→rename→chmod,
and an idempotent sentinel managed-block scan/replace. This is shared infrastructure the
first end-to-end identity slice (02-06) establishes; SSH config, gitconfig fragments,
includeIf blocks, keys, and allowed_signers all write through here. No package may call
`os.WriteFile` directly.

Purpose: SAFE-01/02/03 + KEY-02 are enforced in one place so no mutation path can skip
backup, atomicity, or correct permissions (the project's core safety promise).
Output: `internal/filewriter/filewriter.go`, `internal/filewriter/block.go`, and their
test files (replacing the stub test).
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| caller → filewriter | content + target path supplied by other gitid packages (trusted in-process), but paths point at user secret files (`~/.ssh/config`, private key) |
| filewriter → filesystem | the only place that mutates `~/.ssh` / `~/.gitconfig` |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-01 | Information Disclosure | backup file of private key/config | mitigate | `os.Chmod(backup, 0600)` immediately after copy; backup inherits restrictive mode, never world-readable |
| T-02-02 | Denial of Service | partial/crashed write corrupts `~/.ssh/config` (SSH lockout) | mitigate | atomic temp→fsync→rename; on error remove temp and leave target untouched (restore path) |
| T-02-03 | Information Disclosure | key/config written world-readable via umask | mitigate | explicit `os.Chmod(target, mode)` after rename — 0600 key/config, 0644 .pub; never rely on umask |
| T-02-04 | Tampering | predictable temp name race | mitigate | `os.CreateTemp(dir, "gitid-*.tmp")` unique name in same dir, never a fixed `.tmp` |
| T-02-05 | Tampering | gosec G304/G306 file-path/perm findings | mitigate | paths are gitid-owned managed paths; document trust + keep modes explicit so gosec passes (make lint hard-fail) |
</threat_model>
