<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| gitid → `~/.ssh/config` | gitid mutates the user's SSH config (a lockout-sensitive file) |
| rendered config → `ssh` resolution | the directives gitid writes drive which key ssh offers |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-13 | Spoofing | wrong key offered to provider (too-many-keys) | mitigate | every alias block emits `IdentitiesOnly yes` + explicit `IdentityFile` (SSH-01) |
| T-02-14 | Denial of Service | `UseKeychain` without `IgnoreUnknown` breaks `ssh -G` on Linux | mitigate | emit `IgnoreUnknown UseKeychain` before the Apple-only directive (Pitfall 4) |
| T-02-15 | Tampering | `Host *` first lets wildcard override specific aliases | mitigate | renderer/writer always orders `_global` block LAST (Pitfall 5) |
| T-02-16 | Denial of Service | non-atomic write corrupts config | mitigate | writer delegates to `filewriter.Write` (atomic temp→rename, backup) — never os.WriteFile |
| T-02-17 | Tampering | blind append duplicates host blocks on re-run | mitigate | idempotent sentinel `ReplaceBlock`; foreign content byte-identical (SAFE-02) |
</threat_model>
