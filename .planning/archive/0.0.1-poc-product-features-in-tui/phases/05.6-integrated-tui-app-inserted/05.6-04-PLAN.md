<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| inline edit commit → ~/.ssh, ~/.gitconfig, fragment, allowed_signers | A structural edit re-points config; a non-structural edit rewrites the fragment — all are mutating writes |
| baseline edit → ~/.gitconfig + ~/.gitignore_global | Global options edits mutate the baseline managed blocks |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05.6-10 | Tampering | structural inline edit writes without proving | mitigate | isStructuralField routes alias/hostname/port/match to the prove-before-write loop; the write gate (confirmActive after both phases PASS) holds before identity.Update (D-07) |
| T-05.6-11 | Tampering | any edit writes without backup/confirm | mitigate | All writes go through identity.UpdateDeps / gitconfig baseline writers (filewriter backup); every commit shows a confirm; Esc is the safe default (SAFE-01/02/03) |
| T-05.6-12 | Tampering | skip-test path writes silently | mitigate | Skip requires an explicit second confirm and surfaces a "! written without authentication verification [warning]" notice (UI-SPEC) |
| T-05.6-13 | Information Disclosure | match preview leaks no secrets | accept | Preview shows only the includeIf condition + fragment path (user's own config); no key material |
| T-05.6-SC | Tampering | go installs | accept | No new dependencies (RESEARCH Package Legitimacy Audit: N/A) |
</threat_model>
