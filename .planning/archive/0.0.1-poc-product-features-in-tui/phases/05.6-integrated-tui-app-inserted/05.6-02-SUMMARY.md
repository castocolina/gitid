# Phase 05.6 Plan 02: Two-Pane Persistent Shell Summary

Vertical slice 1 — the thinnest runnable integrated app shell with real live wiring. Replaces the Plan 01 rootModel stub with a persistent two-pane layout (header + sidebar + main + footer), a sidebar that reconstructs identities from disk via the ReadFile seam, a help overlay, and a Ctrl+P command palette. All 8 Plan 01 RED scaffolds are now GREEN.

## What Was Built

### Tasks 1+2: Two-pane shell + sidebar + help/palette modals (one logical commit, 918c055)

**tui/model.go (full rewrite):**

- `viewKind` enum: `identitiesView` / `healthView` / `globalOptionsView`
- `modalKind` enum: `noModal` / `helpModal` / `paletteModal`; others reserved for Plans 04-06 with documented TODO
- `rootModel` struct: `activeView`, `activeModal`, `sidebar sidebarModel`, `sidebarCollapsed`, `focused`, `toast`/`toastStyle`, `palette paletteModel`, `deps tuiDeps`
- `Init()`: returns `sidebar.refresh(deps)` — live data on launch, satisfying D-16 (anti-blindspot)
- `View()`: returns `tea.View{AltScreen: true}` calling `renderContent()` — no `tea.WithAltScreen()` (Pitfall 1)
- `renderContent()`: plain "Terminal too small — resize to at least 80x24" guard at <80×24 (no lipgloss); persistent layout for ≥80×24; modal compositing via `placeOverlay` over `StyleDimmed` base
- `renderPersistentLayout()`: `JoinVertical(Left, header, body, footer)` where body is `JoinHorizontal(Top, sidebar, "│", main)` in normal mode or main-only in collapsed mode
- `renderHeader()`: "gitid" + view tabs (active = StyleTabActive, inactive = StyleTabInactive) + right-aligned toast/badge slot
- `renderFooter()`: per-view bold key hints always rendered (closes G-02/G-04); toggle sidebar hint appears early in list in collapsed mode so it survives ANSI-aware truncation
- `handleKey()`: global keys at root (q, ?, ctrl+p, 1/2/3, \\, tab, arrow keys); modal routing first (Pitfall 9); unhandled action keys (a/e/c/d/R/x) emit "coming next" toast (T-05.6-05)
- `truncateString()`: ANSI-aware using `lipgloss.Width` prefix scan; rune-level byte counting was incorrectly truncating styled strings

**tui/sidebar.go (new):**

- `unmanagedEntry` struct: `shortName` (first 12 chars of Host alias or key filename)
- `sidebarModel`: `accounts []identity.Account`, `unmanaged []unmanagedEntry`, `selected int`, `badges map[string]doctor.Severity`, `width/height int`
- `newSidebarModel()`: zero-value with badge map initialized; caller seeds via `refresh()`
- `refresh(deps tuiDeps) tea.Cmd`: re-reads `~/.ssh/config` and `~/.gitconfig` via `deps.doctor.ReadFile` seam, calls `identity.Reconstruct`, falls back to `deps.doctor.Identities` when reconstruction yields zero accounts (test mode without real disk files) — this is the D-16 live-wiring path
- `buildUnmanaged()`: class-3 orphan key detection from `AllSSHHostIdentityFiles` set vs `KeyPaths` (unused-key orphans; class-1/class-2 are managed blocks, not hand-written)
- `view(width, height int, focused bool) string`: "Identities" section header + rows; "Unmanaged" section only when non-empty; pads to `height` lines
- `renderAccountRow()`: `› name  provider  (N) [badge]` where badge defaults to `✓` (Plan 03 fills `badges` map)

**tui/help.go (new):**

- `renderHelpModal(termW int) string`: stateless; renders UI-SPEC § Help Overlay shortcut table via `modalBox()`; 16 key bindings covering all views and actions

**tui/palette.go (new):**

- `paletteItem`: `label string`, `action string`
- `paletteModel`: `textinput.Model` filter, `[]paletteItem` items/filtered, `cursor int`, `activated bool`
- `newPaletteModel()`: seeds `paletteViewItems` (3 view-switch items); action items wired when Plans 04-06 land
- `applyFilter()`: `fuzzy.Find` (sahilm/fuzzy v0.1.1, indirect dep) with substring fallback
- `update()`: handles up/down cursor, Enter activation, delegates to textinput for filter keys

**tui/messages.go (modified):**

- `refreshSidebarMsg` extended with `accounts []identity.Account` and `unmanaged []unmanagedEntry` fields — the root model's Update handler populates `sidebar.accounts` and `sidebar.unmanaged` from these

## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 3 - Blocking] Tasks 1+2 collapsed into one logical commit per CLAUDE.md**
- **Found during:** Task 1 (model.go references `sidebarModel` before sidebar.go exists)
- **Issue:** `rootModel` struct field `sidebar sidebarModel` and `newRootModel` call to `newSidebarModel()` require sidebar.go to compile. Without it, model.go cannot be compiled or committed alone.
- **Fix:** Implemented Task 1 and Task 2 together as one logical commit (918c055). CLAUDE.md: "One coherent change — its implementation, its tests, and its docs — is a single commit."
- **Files modified:** tui/model.go, tui/sidebar.go, tui/help.go, tui/palette.go, tui/model_test.go, tui/sidebar_test.go, tui/messages.go
- **Commit:** 918c055

**2. [Rule 1 - Bug] Footer truncation cut "toggle sidebar" hint before it appeared**
- **Found during:** TestSidebarCollapse (first run after toggle hint was added last in list)
- **Issue:** `truncateString` used `utf8.RuneLen` which counted ANSI escape bytes as visible columns. With styled hints, the function truncated after ~23 visible chars (the ANSI bytes consumed the budget). Additionally the hint was the 7th of 9 items and got truncated.
- **Fix 1:** Moved `\\  toggle sidebar` hint to appear early in the collapsed footer list (after `tab focus`), before per-view action hints, so it survives even aggressive truncation.
- **Fix 2:** Rewrote `truncateString` to use `lipgloss.Width` prefix scan for ANSI-aware visible-width measurement.
- **Commit:** 918c055

**3. [Rule 1 - Bug] `max` function redefined built-in (revive lint error)**
- **Found during:** `make lint` on palette.go
- **Issue:** `func max(a, b int) int` redefines the Go 1.21+ built-in `max`. golangci-lint v2 revive linter flags this.
- **Fix:** Removed custom `max` helper; used inline `len(m.filtered) - 1` at the call site (Go built-in `max` would also work but inline is clearer).
- **Commit:** 918c055
