<objective>
Vertical slice 1: the thinnest **runnable** integrated app shell. Replace the
screen-stack `rootModel` (D-15) with a persistent two-pane layout — header, always-
visible sidebar (managed identities + read-only unmanaged section), master/detail
main pane, and an always-rendered footer — plus the cross-view chrome that closes
the Phase 5 UAT gaps: `1/2/3` + `Ctrl+P` view switching, `?` help overlay, empty
state, and responsive collapse.

After this plan, a user who runs `gitid` (no args) on a TTY sees the real integrated
app: identities in the sidebar, a detail/empty pane on the right, a bold footer, and
working view-switch/help/palette navigation. Health and Global Options render as
"loading"/placeholder panes (filled by Plans 03/06); the create wizard, inline edit,
delete, rotate, and copy are stubbed to "coming next" toasts (filled by Plans 04-06).
This is a complete, navigable shell — not a horizontal layer.

Purpose: Establish the persistent layout + navigation spine every later slice folds
into, and prove the REAL wired entry launches and is navigable (D-16).
Output: rewritten `tui/model.go`, new `tui/sidebar.go`, `tui/help.go`, `tui/palette.go`.
</objective>
