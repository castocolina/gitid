## Shared Patterns

### Pattern A: tea.Model Interface (v2 contract)

**Source:** `tui/model.go` lines 1–152
**Apply to:** `tui/model.go` (new root), all sub-models

Root model returns `tea.View` (not `string`) from `View()`. Sub-models return `string` from their `view()` (lowercase). AltScreen is set on the returned `tea.View` struct — NOT via `tea.WithAltScreen()` (v1 only).

```go
// tui/model.go lines 145-152 — KEEP EXACTLY as is in the new root model
func (m rootModel) View() tea.View {
    if len(m.stack) == 0 {
        return tea.NewView("")
    }
    v := tea.NewView(m.stack[len(m.stack)-1].view())
    v.AltScreen = true
    return v
}
```

New root model `View()` changes only the content expression:
```go
func (m rootModel) View() tea.View {
    v := tea.NewView(m.renderContent()) // renderContent handles modal vs layout
    v.AltScreen = true
    return v
}
```

### Pattern H: tea.Cmd wrapping blocking I/O

**Source:** `tui/prove.go` lines 118–127, `tui/copy.go` lines 13–18
**Apply to:** `tui/wizard.go`, `tui/confirm.go`, `tui/copy.go`, `tui/health.go`

```go
// tui/prove.go lines 118-127 — goroutine cmd pattern
func runPreWriteCmd(preWrite func(...) tester.Result, keyPath, hostname string, port int) tea.Cmd {
    return func() tea.Msg {
        fn := preWrite
        if fn == nil {
            fn = tester.PreWrite  // defensive fallback when seam is nil
        }
        result := fn(keyPath, hostname, port)
        return preWriteResultMsg{result: result}
    }
}

// tui/copy.go lines 13-18 — same pattern for clipboard
func runClipboardCopyCmd(pubLine string) tea.Cmd {
    return func() tea.Msg {
        err := clipboard.Copy(pubLine)
        return clipboardResultMsg{err: err}
    }
}
```

### Pattern I: screenModel interface (renamed to sub-model convention)

**Source:** `tui/model.go` lines 11–28
**Apply to:** `tui/sidebar.go`, `tui/detail.go`, `tui/health.go`, `tui/globalopts.go`, all modal files

In Phase 5.6 the `screenModel` interface concept is kept for sub-models, but renamed since they are no longer "screens". Each sub-model implements `update(msg tea.Msg) (SubModelType, tea.Cmd)` and `view() string`. The `initializer` interface pattern (init on activation) becomes the `init()` method called when a sub-model becomes active.

```go
// tui/model.go lines 15-28 — interface contracts (adapt, not copy verbatim)
type screenModel interface {
    update(msg tea.Msg) (screenModel, tea.Cmd)
    view() string
}

type initializer interface {
    initScreen() (screenModel, tea.Cmd)
}
```

```go
// SeverityGlyph returns the glyph for the given severity.
// Warning uses "!" — NOT "✗" (Eval #2 fix, D-10).
func SeverityGlyph(s doctor.Severity, asciiMode bool) string {
    if asciiMode {
        switch s {
        case doctor.SeverityCritical, doctor.SeverityError:
            return "FAIL"
        case doctor.SeverityWarning:
            return "!"
        default:
            return "i"
        }
    }
    switch s {
    case doctor.SeverityCritical, doctor.SeverityError:
        return "✗"
    case doctor.SeverityWarning:
        return "!"  // DISTINCT from "✗" — hard rule
    default:
        return "~"
    }
}
```
