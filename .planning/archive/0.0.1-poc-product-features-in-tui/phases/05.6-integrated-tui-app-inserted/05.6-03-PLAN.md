<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| TUI fix action → ~/.ssh and ~/.gitconfig | `x`→confirm→core fix mutates permissions/managed blocks; must be gated by an explicit confirm |
| doctor goroutine → TUI | Async family/fix goroutines can panic; must not crash the program |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05.6-06 | Tampering | in-app fix writes without confirm | mitigate | Fix runs ONLY after Enter on the fix confirm modal; Esc is the safe default; consequence (the exact fix command) shown before the prompt (D-09) |
| T-05.6-07 | Tampering | fix bypasses safe-write chokepoint | mitigate | runFixCmd calls only doctor.Deps.FixPerm/RemoveBlock/AddWiring, which route through filewriter (backup) as wired in buildTUIDoctorDeps; no direct os.WriteFile |
| T-05.6-08 | Denial of Service | family/fix goroutine panic | mitigate | recover() wrap in makeFamilyCmd and runFixCmd converts panics into error messages (ported pattern) |
| T-05.6-09 | Spoofing/Information Disclosure | nil CheckFn silently passing (D-16 blindspot) | mitigate | makeFamilyCmd nil-guard returns an explicit error; the wiring nil-guard test (Plan 06) covers all 8 CheckFn fields |
| T-05.6-SC | Tampering | go installs | accept | No new dependencies (RESEARCH Package Legitimacy Audit: N/A) |
