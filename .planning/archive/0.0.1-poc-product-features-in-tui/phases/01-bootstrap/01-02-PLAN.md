<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| `make setup-env` → remote installer scripts | `setup-env` downloads and executes the golangci-lint `install.sh` over the network — remote code execution surface |
| Makefile recipes → shell | Recipes invoke the shell; any `eval` / unquoted interpolation is an injection surface |
| gosec config → Phase-2 filewriter | gosec is wired NOW so the Phase-2 atomic-write/permission code (PITFALLS P6, P7) is caught from its first commit |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-04 | Tampering | golangci-lint `install.sh` download in `make setup-env` | mitigate | Pin the exact version `v2.12.2` in the installer invocation (not `latest`); use the official `golangci-lint.run/install.sh` over HTTPS with `-sSfL`; install into `$(go env GOPATH)/bin`. Documented, version-pinned install is the project's chosen mechanism (STACK.md). The `[ASSUMED]`/`[SUS]` package-legitimacy gate does not apply (no npm/pip/cargo registry package; this is a vendor install script). |
| T-01-05 | Elevation of Privilege | Makefile shell recipes | mitigate | No `eval`; no unquoted expansion of external input; all tool paths come from `$(go env GOPATH)`. Recipes call fixed binaries with literal args. |
| T-01-06 | Information Disclosure / Tampering | gosec coverage of future file writes | mitigate | gosec enabled in `.golangci.yml` (G304 file-path, G306 perms, G204 exec) so the Phase-2 `filewriter` cannot ship with insecure `~/.ssh` permissions or shell-exec injection undetected. This is the block-on requirement satisfied at config time. |
| T-01-SC | Tampering | tool installers (golangci-lint, gosec, pre-commit) | mitigate | golangci-lint pinned to a fixed version via official installer; gosec and pre-commit are well-known tools. No application npm/pip/cargo dependency is added here. pre-commit (pip) install is delegated to 01-03 where its install command lives; flagged there. |
</threat_model>
