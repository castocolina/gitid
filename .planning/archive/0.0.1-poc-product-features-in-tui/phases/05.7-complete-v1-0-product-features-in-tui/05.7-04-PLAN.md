<objective>
Build the UI-free `internal/uploader` package (AUTOUP-01 core) test-first:
detect `gh`/`glab` on PATH, probe their auth status, and upload a single public
key via an explicit `os/exec` arg slice — `gh ssh-key add <pub> --title <t> --type
authentication|signing` and `glab ssh-key add <pub> -t <t> --usage-type <usage>`.
This is detect-then-prompt only (D-11): the package never drives an interactive
login, never auto-uploads, and never gates create or the test loop — callers
decide per-key with explicit confirmation (D-12).

Purpose: A thin, fully-injected wrapper that the copy modal + wizard step 3
(Plan 07) and the `gitid identity copy --upload-keys` CLI (Plan 06) compose.
Keeping exec behind `Deps.RunCmd` makes arg-slice construction unit-testable
without invoking gh/glab; the real binaries are exercised via PATH-stub fakes in
the Plan 01 e2e harness (D-13).
Output: GREEN `internal/uploader` package replacing the Plan 01 RED stub.
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| pubPath + title → gh/glab subprocess | Caller-supplied path/title cross into a subprocess; could carry metacharacters |
| gh/glab auth state → upload decision | Auth status governs whether prompts are shown; never a login flow |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05.7-04-01 | Tampering | pubPath/title → gh/glab subprocess | mitigate | UploadKey builds an explicit arg slice passed via Deps.RunCmd; never sh -c; the live exec closure (Plan 06) uses the same arg-slice + gosec G204 nolint; Task 2 grep gates + metachar test |
| T-05.7-04-02 | Information Disclosure | private key uploaded | mitigate | Only the `.pub` path is ever passed; UploadKey signature takes pubPath; never the private key (asserted by passing only *.pub fixtures) |
| T-05.7-04-03 | Spoofing | driving an interactive gh/glab login | accept (out of scope) | Package only reads `auth status`; on AuthNotLoggedIn it returns status — caller falls back to manual instructions (D-11/D-12) |
| T-05.7-04-SC | Tampering | npm/pip/cargo installs | accept | No package installs; gh/glab are optional external CLIs detected at runtime, not added to go.mod |
</threat_model>
