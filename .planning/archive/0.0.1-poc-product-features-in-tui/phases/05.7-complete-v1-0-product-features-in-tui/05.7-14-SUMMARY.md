# Phase 05.7 Plan 14: Core Provisional Block Lifecycle Summary

One-liner: Transactional provisional SSH Host block lifecycle (distinct sentinel, staged-key faithful, promote/drop never-corrupt) + EffectiveAlias WYSIWYG blank-alias rule fixes UAT G-5 BLOCKER.

## What Was Built

### Task 1: filewriter provisional sentinel + splice helpers

**`internal/filewriter/block.go`** refactored to extract three sentinel-parameterized private cores:

- `listBlocksWith(content, beginPrefix, endPrefix)` — shared scan core
- `removeBlockWith(content, name, beginPrefix, endPrefix)` — shared remove core
- `replaceBlockWith(existing, name, blockBody, beginPrefix, endPrefix)` — shared replace core

The exported managed helpers (`ListBlocks`, `RemoveBlock`, `ReplaceBlock`) are now thin wrappers passing `BeginPrefix`/`EndPrefix` — output is byte-identical to the prior implementation, verified by running the unchanged existing test suite (`block_test.go`, `block_list_test.go`, `block_prepend_test.go`).

**`internal/filewriter/provisional.go`** exposes:

- `ProvisionalBeginPrefix = "# BEGIN gitid provisional: "` — DISTINCT from `BeginPrefix`
- `ProvisionalEndPrefix = "# END gitid provisional: "` — DISTINCT from `EndPrefix`
- `ReplaceProvisionalBlock(existing, name, body)` — appends/replaces provisional block
- `RemoveProvisionalBlock(content, name)` — removes only provisional sentinel range
- `ListProvisionalBlocks(content)` — returns provisional blocks in file order

Mutual exclusion verified in test `TestListProvisionalBlocks_MutualExclusion`: `ListBlocks` returns only managed blocks, `ListProvisionalBlocks` returns only provisional blocks.

Full round-trip test `TestProvisionalRoundTrip_WriteListRemove` asserts that after write→list→remove, the content is byte-identical to the original base (managed block + foreign content untouched).

### Task 2: sshconfig provisional lifecycle

**`internal/sshconfig/provisional.go`** adds four functions mirroring the `Write` chokepoint pattern:

- `WriteProvisional(configPath, name, hostBlock string) (backupPath string, err error)` — reads existing (missing = empty), composes via `ReplaceProvisionalBlock`, Parse-validates, writes via `filewriter.Write`
- `Promote(configPath, name, managedHostBlock string) (backupPath string, err error)` — ONE composed write: `RemoveProvisionalBlock` then `ReplaceBlock` on same in-memory bytes, Parse-validates, single `filewriter.Write` (atomic swap, no half-state T-05.7-14-05)
- `DropProvisional(configPath, name string) (backupPath string, err error)` — removes provisional block; missing file → `("", nil)`; absent block → idempotent write-back (still backs up)
- `ListProvisional(content []byte) []string` — returns provisional block names in file order

All writes go through `filewriter.Write` (atomic temp→rename, 0600, timestamped backup). Parse failure before any write → error + no file mutation (T-05.7-14-02).

Tests drive real temp config files on disk and assert:
- `WriteProvisional` adds a `# BEGIN gitid provisional:` block with the staged key path
- `Promote` produces a `# BEGIN gitid managed:` block with the final key path and NO provisional marker
- `DropProvisional` removes the provisional block and preserves managed + foreign content byte-for-byte
- Each call produces a timestamped backup
- `TestPromote_BackupContainsProvisional`: backup snapshot is the pre-promote state (contains provisional marker)

### Task 3: identity EffectiveAlias + provisional lifecycle

**`internal/identity/identity.go`** extended (additive):

```go
WriteProvisionalSSH func(name, hostBlock string) (backupPath string, err error)
PromoteSSH          func(name, hostBlock string) (backupPath string, err error)
DropProvisionalSSH  func(name string) (backupPath string, err error)
```

Three new fields on `identity.Deps`. All existing callers compile unchanged (nil zero-value fields are not referenced by existing code paths).

**`internal/identity/provisional.go`** adds:

- `EffectiveAlias(aliasField, provider string) string` — trimmed alias when non-empty; blank → provider host (dotted provider verbatim lowercased; undotted → `+".com"`). Never returns an invented `<name>.<provider>` suffix.
- `PersistSSHProvisional(in, staged, deps)` — renders Host block with `staged.TempPrivatePath`, calls `deps.WriteProvisionalSSH`; does NOT call PersistKey, WriteSSH, PromoteSSH, or WriteGitconfig
- `PromoteSSH(in, staged, deps)` — calls `deps.PersistKey` FIRST (T-05.7-09-04 ordering), then renders managed Host block with `staged.FinalPrivatePath` and calls `deps.PromoteSSH`; skips PersistKey when `staged.PrivPEM == nil`
- `DropProvisionalSSH(in, deps)` — calls only `deps.DropProvisionalSSH`; does not delete staged key

Call-log fake tests assert the call-ordering and seam isolation for all three functions.
