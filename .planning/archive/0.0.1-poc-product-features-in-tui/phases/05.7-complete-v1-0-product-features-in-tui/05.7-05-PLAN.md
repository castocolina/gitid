<objective>
Complete the deferred Phase 5.6 D-06 match-strategy selector (the project's
long-open gitdir-vs-hasconfig divergence): keep `gitdir` as the DEFAULT for new
identities (D-02) but expose `hasconfig` and `both` as selectable options, with
the existing live `includeIf` preview, on BOTH surfaces (D-10 parity). The
`hasconfig` value auto-derives from the alias as `git@<alias>:*/**` (recipe form,
D-03) and is editable before write; `both` writes two OR-applied includeIf blocks.

Which surface closes the match e2e stubs (REVIEWS.md #6): `e2e/match_e2e_test.go`
TestMatchStrategy_Gitdir/HasConfig/Both drive `gitid identity add` over STDIN —
they feed a strategy line ("1"/"2"/"3") and the strategy-dependent value line
BETWEEN the port prompt and the passphrase prompt. The CLI `gatherCreateInput`
(cmd/gitid/add.go line 388) ALREADY calls `promptMatchStrategy` in exactly that
position. So Task 1 turns the e2e stubs GREEN by VERIFYING and, if needed,
REPAIRING the interactive stdin picker (prompt order + the FIX-CREATE-01 PASS
auto-persist that dropped the trailing confirm line), NOT by adding a flag. The
`--match gitdir|hasconfig|both` flag is the ADDITIONAL non-interactive parity
surface — it does not feed the stdin picker the stubs exercise.

Purpose: The core (`gitconfig.MatchGitdir`/`MatchHasconfig`, `promptMatchStrategy`,
`buildMatches`) already supports all three kinds; the CLI picker already exists.
The remaining gaps are: (1) the TUI wizard still shows a read-only gitdir-only
label, and (2) a non-interactive `--match` flag for scripting parity. This plan
closes both and proves the existing e2e stubs are GREEN.
Output: GREEN match e2e (stdin picker) + TUI selector + CLI `--match` flag.
</objective>
