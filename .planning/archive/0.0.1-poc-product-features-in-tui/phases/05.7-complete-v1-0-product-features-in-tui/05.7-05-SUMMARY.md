# Phase 05.7 Plan 05: Match-Strategy Selector Summary

**One-liner:** Gitdir/hasconfig/both match-strategy selector added to CLI (--match flag with shell completion) and TUI wizard (expanded radio selector with live includeIf preview via RenderIncludeIf).

## What Was Built

### Task 1: e2e stubs GREEN + --match parity flag on add/update

The three pre-existing RED e2e stubs (`TestMatchStrategy_Gitdir/HasConfig/Both`) were already GREEN at plan start — the interactive stdin picker (`promptMatchStrategy`) was correctly wired at `gatherCreateInput` line 388 in Phase 05.5-04. Plan 05 verifies this and adds the non-interactive parity surface:

- **`matchFromFlag(s string) (string, error)`** in `cmd/gitid/match.go`: maps `gitdir/""→"1"`, `hasconfig→"2"`, `both→"3"`; returns error for any other value (T-05.7-05-02 whitelist).
- **`--match` flag** added to `addFlags` (`cmd/gitid/add.go`) and `updateFlags` (`cmd/gitid/update.go`) with `RegisterFlagCompletionFunc` registering `{gitdir, hasconfig, both}` completions (shell tab-completion).
- `gatherCreateInput` and `runIdentityUpdate` extended: when `--match` is set (and `--gitdir`/`--url` are absent), `matchFromFlag` derives the strategy number and `buildMatches` builds the match list using sensible defaults for the missing sub-fields.
- Tests: `TestMatchFromFlag_Gitdir`, `TestMatchFromFlag_Empty`, `TestMatchFromFlag_Hasconfig`, `TestMatchFromFlag_Both`, `TestMatchFromFlag_Unknown`, `TestMatchFromFlag_CaseInsensitive`.

### Task 2: TUI wizard expanded match-strategy selector

Replaced the read-only "gitdir — match repos by folder" label in the wizard form with a full interactive selector:

- **`matchStrategy` enum**: `strategyGitdir` (default, D-02), `strategyHasconfig`, `strategyBoth`.
- **`defaultHasconfigPattern(alias string) string`**: derives `git@<alias>:*/**` (recipe canonical form, D-03); returns `""` for empty alias.
- **`liveIncludeIfPreview(strategy, name, _, gitdirVal, hasconfigVal)`**: pure helper that calls `gitconfig.RenderIncludeIf` for ALL three strategies — no hand-built `[includeIf "..."]` strings (T-05.7-05-01 grep gate: 7 `RenderIncludeIf` calls, 0 `fmt.Sprintf` includeIf).
- **`createWizardModel` extended** with `matchSel matchStrategy`, `matchGitdirVal textinput.Model`, `matchHasconfigVal textinput.Model` fields; `syncMatchInput()` bridges `matchSel` to `inputs[6]` for `buildCreateInput` backward compatibility.
- **`handleKey` on `fieldMatch`**: up/down cycle gitdir→hasconfig→both; typing forwards to active sub-field; `syncMatchInput` called on strategy change.
- **`renderMatchSelector`**: collapsed form (not focused) shows summary line + compact `includeIf preview:` block; expanded form (focused) shows 3-option radio buttons with sub-fields, live `Preview:` block (UI-SPEC), OR-applied footer for `both`, recipe-alignment note.
- **`buildCreateInput`** rewritten to derive matches from `matchSel` + sub-field values; gitdir remains the default (D-02).
- Tests: `TestDefaultHasconfigPattern`, `TestDefaultHasconfigPatternEmpty`, `TestLiveIncludeIfPreview_Gitdir`, `TestLiveIncludeIfPreview_Hasconfig`, `TestLiveIncludeIfPreview_Both`, `TestWizardMatchSelectorNavigation`, `TestWizardMatchValidationCopyPresent`.
