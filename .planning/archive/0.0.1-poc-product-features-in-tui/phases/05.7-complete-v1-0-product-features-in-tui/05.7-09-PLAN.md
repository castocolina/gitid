<objective>
CORE REFACTOR (UI-free, TDD) for the staged wizard restructure. Two changes, both
in the UI-free internal/identity package so the CLI and the new staged TUI wizard
share one write path (D-10 parity):

1. Centralize the provider -> alt-SSH-host mapping in ONE UI-free helper
   (identity.DefaultHostname / identity.DefaultPort) so the CLI and the TUI share it
   and the literal alt-SSH strings live in exactly one place. Complete the recipe-
   correct mapping (add the missing bitbucket endpoint).
2. Split the monolithic identity.PersistAll into two composable, independently
   callable steps so the staged wizard can write PER LEG:
     * PersistSSH       -- LEG 1: persist the key (when PrivPEM != nil) + write the
                           ~/.ssh/config Host block. Returns the SSH backup path.
     * PersistGitconfig -- LEG 2: write the ~/.gitconfig.d/<name> fragment + the
                           includeIf block + ~/.ssh/allowed_signers. Returns the
                           gitconfig + allowed_signers backup paths.
   PersistAll is rewritten as a THIN composition: PersistSSH -> PersistGitconfig ->
   Resolved, so the existing CLI single-shot create flow (cmd/gitid/add.go
   runCreateNew) keeps byte-identical behavior.

This plan is the FOUNDATION for the staged wizard (Plans 10/12/13). It is UI-free
(no charm/bubbletea imports) and runs in parallel with the doctor plan (11) in
wave 5 -- zero file overlap (this = internal/identity + cmd/gitid/add.go; 11 =
doctor + tui/deps + tui/health).

Closes (foundation for): G-1, G-2, G-3 (the staged wizard built on this dissolves
them). Resolves the recipe alt-SSH divergence per user decision "Align + keep 22
fallback". Implements GIT-02 (recipe-aligned SSH config) at the core layer.

Purpose: The staged wizard must write LEG 1 (SSH config) at the end of Screen 2 and
LEG 2 (gitconfig) at the end of Screen 3. The current PersistAll writes all four
artifacts in one shot, which cannot express progressive-per-leg writes. Splitting it
here -- in the UI-free core, test-first -- gives both the CLI and the TUI one
authoritative write path with no logic duplicated in tui/.
Output: A UI-free alt-SSH mapping shared by CLI+TUI; two composable leg-write steps
(PersistSSH, PersistGitconfig); PersistAll as their thin composition with unchanged
CLI behavior; the staged-key model preserved.
</objective>
