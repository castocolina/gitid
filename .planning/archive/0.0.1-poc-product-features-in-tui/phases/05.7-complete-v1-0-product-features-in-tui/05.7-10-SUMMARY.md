# Phase 05.7 Plan 10: Staged Wizard Screen 1 + Viewport Fix Summary

One-liner: Staged wizard Screen 1 with always-visible alt-SSH fields (name/algo/provider/alias/hostname/port/folder), live RenderHostBlock preview, and viewport-aware modal that never silently clamps overflow.

## What Was Built

### Task 1: Viewport-aware modal — boundModalToViewport (G-1 overflow guard)

**Root cause**: `renderContent` in `model.go` computed `mh = lipgloss.Height(modalContent)` POST-render and passed it to `modalOrigin`, yielding a negative `y` for tall modals. `placeOverlay` then silently dropped any `fgLine` whose `bgRow >= len(bgLines)` — rows just vanished.

**Fix applied** (Option B: measured layout + indicator):
- `boundModalToViewport(modal, available, scrollOffset)` in `overlay.go`: clips the modal to `available` rows with an explicit `↓ more` / `↑↓ scroll` indicator on the last visible row. When the modal fits, returns it unchanged (no regression to TestPlaceOverlayCompositesModal).
- `renderContent` in `model.go`: bounds `modalContent` to `m.height - 4` before `modalOrigin` / `placeOverlay`, so `y` is never negative and the overlay never falls off-screen unrecoverably.

**Rationale for Option B over A (full scroll)**: The staged wizard (Task 2) keeps each screen small by design. The guard exists for degenerate tiny terminals. A full per-modal scroll tracker would require threaded focus-position state across every modal sub-model — disproportionate complexity for a safety net.

**Tests**:
- `TestViewportBoundModalLinesReachable`: asserts every modal line is reachable across scroll positions; fits-on-screen modals unchanged.
- `TestRenderContentViewportAwareModal`: renderContent doesn't panic with wizard open on 80x24; output bounded to 24 lines.

### Task 2: Staged-flow scaffolding + Screen 1 SSH-identity form

**wizardScreen enum** added: `screenSSHIdentity / screenConnectivity / screenGitConfig / screenReview`. `createWizardModel` carries a `screen` field (default `screenSSHIdentity`). `viewForm` routes by screen: `viewScreen1` for Screen 1, `viewLegacyForm` for others.

**New Screen-1-exclusive fields**:
- `hostnameVal textinput.Model`: pre-filled with `identity.DefaultHostname("github.com")` = `"ssh.github.com"`.
- `hostnameEdited bool`: set to `true` on first user keystroke to the hostname field; while `false`, `refreshHostnameIfUnedited()` auto-tracks provider changes.
- Port default changed from `"22"` to `identity.DefaultPort()` = `"443"` (alt-SSH).

**Screen-1 Tab focus mapping** (`screen1FocusCount = 6`):
| Pos | Field | Backend |
|-----|-------|---------|
| 0 | Identity Name | inputs[0] |
| 1 | Provider | inputs[3] |
| 2 | SSH Alias | inputs[5] |
| 3 | Hostname | hostnameVal (virtual) |
| 4 | Port | inputs[4] |
| 5 | Folder (gitdir) | matchGitdirVal (virtual) |

Key Algorithm (ed25519) is rendered read-only; Git Name/Email/Match/Signing are NOT on Screen 1.

**buildCreateInput** updated:
- `Hostname`: uses `hostnameVal` value (alt-SSH default or user override); removes the `Hostname: provider` literal.
- `Port`: uses `inputs[4]` parsed as int, falls back to `identity.DefaultPort()` (443); removes the hard-coded `port = 22` default.
- For `strategyGitdir`, uses the typed `matchGitdirVal` path (pre-filled `~/git/<name>/`) instead of `identity.DefaultMatch` so Screen-1 folder edits propagate.

**advanceFromForm**: email validation deferred to Screen 3 on `screenSSHIdentity` (non-Screen-1 screens retain early validation for backward compat).

**Tests**: `TestWizardScreen1ShowsSSHIdentityFields`, `TestWizardScreen1AltSSHDefaults`, `TestWizardScreen1HostnameEditable`, `TestWizardScreen1PortEditable`, `TestWizardScreen1FolderEditable`, `TestWizardScreen1TabCycleAllFields`, `TestWizardScreen1HostnameTracksProvider`, `TestWizardScreen1BuildCreateInputHostname`.

### Task 3: Live SSH Host-block preview via RenderHostBlock (G-2 preview half)

**`renderSSHBlockPreview(sb)`** method on `createWizardModel`:
- Derives alias/hostname/port/finalKeyPath from the SAME sources as `buildCreateInput` (preview/write parity, T-05.7-10-02).
- Calls `sshconfig.RenderHostBlock(alias, hostname, port, finalKeyPath, provider)` exclusively — no hand-built Host strings (same discipline as `liveIncludeIfPreview → RenderIncludeIf`).
- `finalKeyPath`: uses `staged.FinalPrivatePath` when set; falls back to `~/.ssh/id_ed25519_<name>` so the preview is meaningful before keygen.
- Guard: rendered only when `name != ""`.

**Wired into `viewScreen1`**: preview section appears after the form rows under a `"Will write to ~/.ssh/config:"` header styled in StyleFaint. Updates live on every `view()` call — alias/hostname/port edits are reflected immediately.

**Tests**: `TestWizardScreen1SSHBlockPreview` (alt-SSH defaults: Host personal.github.com / Hostname ssh.github.com / Port 443 / User git / IdentitiesOnly yes), `TestWizardScreen1PreviewGuardedOnEmptyName`, `TestWizardScreen1PreviewUpdatesOnPortEdit` (Port 22 override; no advisory line), `TestWizardScreen1PreviewUpdatesOnAliasEdit`, `TestWizardScreen1PreviewUsesRenderHostBlock` (staged.FinalPrivatePath appears in IdentityFile line).

## Deviations from Plan

**1. [Rule 2 - Missing behavior] Screen-1 enter-advance trigger at folderFocusIdx() (5) not inputs-last (7)**
- **Found during:** Task 2 implementation
- **Issue:** `TestWizardPersistsOnlyAfterPass` used `w.focusIdx = 7` (last inputs[] index) to trigger `advanceFromForm`. On Screen 1, the advance fires at `screen1FocusCount-1 = 5` (Folder). The test was updated to use `folderFocusIdx()`.
- **Fix:** Updated test to set `w.focusIdx = folderFocusIdx()` (the correct last Screen-1 field). No functional change.
- **Files modified:** `tui/wizard_test.go`

**2. [Rule 1 - Bug] textinput.Update early-returns when !m.focus**
- **Found during:** Task 2 test debugging
- **Issue:** In bubbletea v2, `textinput.Update` returns early if `!m.focus`. Setting `focusIdx = 3` (hostname) without calling `.Focus()` on `hostnameVal` meant typed text wasn't inserted.
- **Fix:** In `handleKeyScreen1`, added `if !m.hostnameVal.Focused() { m.hostnameVal.Focus() }` (and equivalent for `matchGitdirVal`, `inputs[idx]`) before forwarding key events. This is a required correctness fix.
- **Files modified:** `tui/wizard.go`

**3. [Rule 2 - Backward compat] Existing tests (email/signing/match) moved to screenGitConfig**
- **Found during:** Task 2 full suite run
- **Issue:** `TestWizardFormEmailValidation`, `TestWizardSigningToggle`, `TestWizardFormReadableChoices`, `TestWizardMatchSelectorNavigation` tested Screen-3 fields that no longer appear on Screen 1.
- **Fix:** Set `w.screen = screenGitConfig` in each test so the legacy form path is exercised. The behaviors tested (email validation, signing toggle, match selector) are preserved in the legacy path for Plan 13.
- **Files modified:** `tui/wizard_test.go`
