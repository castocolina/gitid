<objective>
DOCTOR: recognize leftover `gitid provisional:` SSH blocks (the distinct sentinel from
Plan 14) and report them as an ADVISORY finding — never as an orphan, error, or identity —
so a crashed/cancelled create run leaves a discoverable but non-destructive trace.

CRITICAL (project memory: doctor-reserved-block-false-positive-loop): a previous gap had
doctor treat a new managed gitconfig block as an identity/orphan and enter a destructive
`--fix` loop. The provisional sentinel MUST be registered/recognized so the Orphans check
and the managed-identity set EXCLUDE provisional blocks; doctor `--fix` must NEVER treat a
provisional block as an identity or an orphan, and must NEVER remove it as part of an
orphan fix. The advisory may offer a SAFE non-destructive cleanup, but provisional blocks
are excluded from the managed-identity set and from orphan detection.

By construction (Plan 14), provisional blocks use `# BEGIN gitid provisional:` and are
invisible to filewriter.ListBlocks (managed scan), which feeds sshconfig.ParseManagedHosts
-> SSHManagedBlockNames. So provisional blocks are ALREADY outside the managed set and the
Orphans Class-1 cross-reference. This plan ADDS the positive advisory recognition and a
regression test that PROVES the exclusion holds end-to-end (a co-resident provisional +
managed block produces only the advisory, never an orphan finding).

Purpose: make a leftover provisional block visible to the user as a safe advisory while
guaranteeing doctor (especially `--fix`) never misclassifies or destructively touches it.
Output: an advisory-only FamilyProvisional check (CheckProvisional) wired into
Run/Families + the CLI + the TUI health view, with a regression test against the
false-positive --fix loop.
</objective>
