# Phase 05.7 Plan 01: Wave 0 Test Scaffolding Summary

**One-liner:** RED Wave 0 scaffold — 3 e2e harness helpers, 3 RED e2e files, 3 production package skeletons (adopter/repoclone/uploader with Deps/enum/sentinel types), 3 RED unit stubs, and RED nil-guard extension for the D-13/D-16 anti-blindspot mechanism.

## What Was Built

### Task 1 — e2e Harness Extension (`e2e/harness_test.go`)

Four new helpers appended to the existing harness, each mirroring the `FakeSSHDir` pattern:

- **`FakeGHDir(t, mode)`** — writes a static fake `gh` shell script (const literal, G204/G306-clean); modes `ok` (auth+upload succeed) and `auth-fail` (not logged in). Sets `GITID_FAKE_GH_MODE`.
- **`FakeGLabDir(t, mode)`** — same pattern for `glab`; sets `GITID_FAKE_GLAB_MODE`.
- **`FakeGitDir(t, mode)`** — fake `git` for clone testing; modes `clone-ok` (mkdir dest, exit 0) and `clone-fail` (exit 128). Sets `GITID_FAKE_GIT_MODE`.
- **`setupLocalBareRepo(t)`** — runs real `git init --bare` into `t.TempDir()`, returns `file://` URL and base name for network-free clone targets.

Verification: `go vet -tags e2e ./e2e/...` exits 0; grep returns 4.

### Task 2 — RED e2e Tests (adopt, addrepo, upload)

Three new files, each `//go:build e2e` + `package e2e`:

- **`e2e/adopt_e2e_test.go`**: `TestAdopt_Migrate` and `TestAdopt_Reference` — drive real `gitid` binary in sandbox HOME, assert `[includeIf]` written and fragment paths correct. RED because `gitid adopt` command doesn't exist yet.
- **`e2e/addrepo_e2e_test.go`**: `TestAddRepo_LocalClone` (local bare repo clone via `file://` URL) and `TestAddRepo_AliasRewrite` (HTTPS→SSH alias rewrite in output). RED because `gitid add repo` doesn't exist yet.
- **`e2e/upload_e2e_test.go`**: `TestUpload_GHOk` / `TestUpload_GHAuthFail` / `TestUpload_GHNotPresent` — use `FakeGHDir` on PATH, assert upload or fallback behavior. RED because `--upload-keys` flag doesn't exist yet.

No `t.Skip` calls in any file — tests fail honestly.
