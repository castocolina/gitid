# Phase 05.7 Plan 12: Screen 2 SSH Test — Full Command Visibility + PersistSSH on Success

One-liner: Screen 2 always shows the full ssh command + staged key path (pre-run/success/failure via PreWriteCommand + renderTestCommandLine), writes LEG 1 via PersistSSH on success or [s] skip, and gives clear no-write guidance with a discoverable offline escape on failure.

## Accomplishments

- Added `tester.PreWriteCommand(keyPath, hostname, port) string` — pure read-only helper returning `exec.Command("ssh", preWriteArgs(...)).String()`, byte-identical to `Result.Command` for the same inputs. Pre-run display never drifts from the actual test command.
- Fixed `initProve` and retry handler to use `proveHostnamePort()` (derives alt-SSH hostname/port from the same sources as `buildCreateInput`) — replaces hard-coded `provider` + `port=22` defaults.
- Added `renderTestCommandLine` helper showing `$ <cmd>` on PASS phases (not only failure); called after Phase-1 and Phase-2 `✓` lines in `viewProve`.
- `viewUpload` now shows the pre-run command via `tester.PreWriteCommand` before Enter is pressed — command visible at every sub-state of Screen 2.
- Routed PASS path and `[s]` skip path through `runWizardPersistSSHCmd` → `identity.PersistSSH` (LEG 1 only): `WriteSSH` fires, `WriteGitconfig` does NOT fire.
- On LEG-1 success the wizard advances to `screenGitConfig` (Screen 3 stub; Plan 13 wires the Git config form).
- Added `renderNoWriteGuidance` showing "Nothing has been written yet.", outcome-keyed WHY line, and two numbered ways forward including a consequence-explicit `[s]` offline escape.
- Updated 'q' quit toast from "Key kept at …" to "Nothing written. Key kept at <path>. Re-run and press [s] to write offline, or upload the key then re-run."
- Removed now-unused `runWizardCreateCmd` (PersistAll); linter clean.

## Decisions Made

- `proveHostnamePort()` extracts the hostname/port derivation logic so `initProve`, the retry handler, and `viewUpload`'s pre-run command all use a single shared source — prevents any drift between display and execution.
- `runWizardPersistSSHCmd` dispatches `identity.PersistSSH` (LEG 1 only). The `[s]` double-confirm gate is preserved exactly (FIX-CREATE-01 / T-05.7-12-01); only the write target changes from `PersistAll` to `PersistSSH`.
- Removed `runWizardCreateCmd` (PersistAll) since both dispatch sites now use `runWizardPersistSSHCmd`. `PersistAll` is still available in the identity package for future plans that need both legs atomically.
- The `wizardCreateResultMsg` handler now advances to `screenGitConfig + wizardStepForm` on success (not `wizardStepWritten`). This is the staged design: a mid-exit after Screen 2 leaves a valid SSH-only identity; Screen 3 (Plan 13) writes LEG 2.
