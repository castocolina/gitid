  <action>
Add three Make variables near the existing SCREENSHOT_* variable block (before the recipes):
`DEMO_WEB_PORT := 45173`, `DEMO_WEB_DIR := $(CURDIR)/.planning/design/mockup-src`, and
`DEMO_WEB_LOG := /tmp/gitid-demo-web.log`. Document each with a short `#` comment: 45173 is
memorable (Vite's default 5173 with a 4-prefix), sits below macOS's ephemeral range (49152+)
so it is stable to bind, and is off 5173 so lsof-kill-by-port cannot collide with another Vite
project (locked decision). DEMO_WEB_DIR uses $(CURDIR) so the recipe is independent of the
caller's working directory, matching the SCREENSHOT_FONT $(CURDIR) precedent.

Add `demo-web` to the `.PHONY` line and add a one-line entry to the header target list at the
top of the file (same `## <target>  <description>` style as screenshot-html), describing it as
"(re)launch the web design mockup dev server (Vite) on the dedicated $(DEMO_WEB_PORT) and open it".

Add the `demo-web` recipe with a `##` doc-comment block above it (matching the screenshot-html
doc style) explaining: kills any previous instance listening on $(DEMO_WEB_PORT), starts Vite in
the background (logs -> $(DEMO_WEB_LOG)) so it survives make returning, waits for the port to bind,
then opens the browser; note it is a design-only surface never part of the Go module or any
build/test/lint gate.

Write the recipe as a SINGLE `/bin/sh` block (leading `@`, lines joined with `; \` so the `cd`
persists and the shell metacharacters force make through a real shell that honors the exported
PATH — the GNU Make 3.81 direct-exec-bare-command quirk documented at the top of the file).
The block must, in order:
  1. Capture `lsof -ti tcp:$(DEMO_WEB_PORT)` PIDs (guard with `2>/dev/null || true`); if non-empty,
     echo which PIDs are being stopped, `kill` them, `sleep 1`, then `kill -9` any survivors still
     bound to the port (guarded so a race where they already exited is not a hard error).
  2. `cd $(DEMO_WEB_DIR)`.
  3. If `node_modules` is absent, echo a notice and run `pnpm install --frozen-lockfile` (never a
     bare `pnpm install` — README workspace rule; the committed pnpm-lock.yaml is the source of truth).
  4. Start the server backgrounded and detached: `nohup pnpm exec vite --port $(DEMO_WEB_PORT)
     --strictPort >$(DEMO_WEB_LOG) 2>&1 &`. Use `pnpm exec vite` (not `pnpm dev`) so the --port/
     --strictPort flags reach Vite directly with no script-arg-forwarding ambiguity; the `dev`
     script is literally `vite`, so this is equivalent. --strictPort makes Vite fail loudly rather
     than silently pick a different port, keeping the opened URL and the bound port in lockstep.
  5. Poll for readiness with a bounded loop (e.g. up to ~40 iterations of `sleep 0.25`), breaking
     as soon as `lsof -ti tcp:$(DEMO_WEB_PORT)` reports the port bound; do not loop unbounded.
  6. Echo and run `open http://localhost:$(DEMO_WEB_PORT)`.
Keep all output prefixed for readability (e.g. a leading `==> demo-web:` echo then indented step
lines), consistent with the setup-env recipe's echo style.
