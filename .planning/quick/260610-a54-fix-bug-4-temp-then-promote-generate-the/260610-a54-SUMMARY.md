# Phase quick-260610-a54 Plan 01: Fix BUG-4 Temp-Then-Promote Generate Summary

Ed25519 key generation refactored to pure in-memory `GenerateMaterial` + hermetic `os.MkdirTemp` staging, with `PersistKey` writing final `~/.ssh` paths via `filewriter` only after gate passes AND `Confirmed` is true.

## What Was Built

### Task 1: keygen.GenerateMaterial + KeyPaths (RED `3ad6cc0` / GREEN `2848cf7`)

Introduced `GenerateMaterial(Params) (Material, error)` — the pure in-memory half of the old `Generate`. Extracts the `ed25519.GenerateKey → MarshalPrivateKey → NewPublicKey → MarshalAuthorizedKey` steps into a function that returns `Material{PrivPEM []byte, PubLine string}` without touching the filesystem. Removed disk-writing `Generate` and unused `privKeyMode`/`pubKeyMode` constants. Added `KeyPaths(dir, algo, identity)` helper for D-06 path convention.

**New tests (all passing):**
- `TestGenerateMaterial_InMemory` — PrivPEM is a parseable OPENSSH PRIVATE KEY block; PubLine has the correct prefix and single-newline
- `TestGenerateMaterial_NoDiskWrite` — dir stays empty even when Dir is set
- `TestGenerateMaterial_Passphrase` — encrypted PEM differs from unencrypted; block type still OPENSSH PRIVATE KEY
- `TestGenerateMaterial_UnsupportedAlgo` — error contains "unsupported algorithm"
- `TestKeyPaths` — D-06 `id_<algo>_<identity>` path and .pub sibling

### Task 2: StagedKey + PersistKey/Cleanup in identity (RED `ab29a7a` / GREEN `08ea719`)

Added `StagedKey` type with `TempPrivatePath`, `FinalPrivatePath`, `FinalPubPath`, `PubLine`, `PrivPEM` fields. Changed `Deps.Generate` signature to return `StagedKey`. Added `Deps.PersistKey` and `Deps.Cleanup` fields. Updated `runPipeline` to:
- Gate on `staged.TempPrivatePath` (BUG-4 fix — runs `ssh -i <temp>` before any `~/.ssh` write)
- Render previews from `FinalPrivatePath`/`FinalPubPath` (never temp)
- Call `PersistKey` only when `staged.PrivPEM != nil` and `in.Confirmed` is true, BEFORE the four writers
- Updated `Reuse` and `AddAccount` to construct `StagedKey` with `TempPrivatePath == FinalPrivatePath` and `PrivPEM nil`
- Updated `Rotate` to call `Generate` + `defer Cleanup`

Updated all fakes (`newFakeDeps`, `newFakeModeDeps`, `fakeDeps` in add_test.go) with non-nil `PersistKey`/`Cleanup` recording fakes.

**New behavioral tests (all passing):**
- `TestCreateDryRun_PersistKeyCountZero` — PersistKey==0, Cleanup==1, all previews non-empty, PreWriteOnly==true
- `TestCreateGateFailure_PersistKeyCountZero` — gate Failure: PersistKey==0, Cleanup==1, no writer ran
- `TestCreateConfirmed_PersistKeyCountOneAndFinalPaths` — PersistKey==1 fires before WriteSSH; res.Key.PrivatePath == FINAL; SSHPreview/FragmentPreview contain FINAL not temp
- `TestCreateGate_UsesTempPath` — PreWrite receives `TempPrivatePath` (not FinalPrivatePath)
- `TestReuseNoPersistKey` — Reuse PersistKey==0 (PrivPEM nil)
- `TestAddAccountNoPersistKey` — AddAccount PersistKey==0 (PrivPEM nil)
- `TestRotatePersistKeyOnConfirm` — confirmed PersistKey==1; gate-failure PersistKey==0

### Task 3: buildDeps rewired (GREEN `f085e5d`)

Rewired `buildDeps` in `cmd/gitid/add.go`:
- **Generate**: `keygen.KeyPaths` → `keygen.GenerateMaterial` → `os.MkdirTemp("", "gitid-key-*")` → `filewriter.Write(tempPriv, PrivPEM, 0600)` → return `StagedKey` with `PrivPEM` carried in-memory
- **PersistKey**: `PrivPEM==nil` guard (no-op for existing keys) → `filewriter.Write(FinalPrivatePath, PrivPEM, 0600)` + `filewriter.Write(FinalPubPath, PubLine, 0644)` — no `os.ReadFile` re-read (no gosec G304)
- **Cleanup**: `PrivPEM==nil || TempPrivatePath==FinalPrivatePath` guard → `os.RemoveAll(filepath.Dir(TempPrivatePath))`
- Removed `keygen.Generate`; updated `derive_test.go` to use `GenerateMaterial` + `os.WriteFile` fixture
