<objective>
Fix BUG-4 (SAFE-03 violation + orphan key on abort) by adopting a temp-then-promote
flow for the two GENERATE paths only — create-new (`identity.Create`) and `Rotate`.
The key is generated to a hermetic temp location, the pre-write `ssh -i <temp>` gate
runs against it, and the key bytes are persisted to `~/.ssh` ONLY after the gate
passes AND `in.Confirmed` is true. Dry-run and gate-failure aborts leave `~/.ssh`
byte-for-byte untouched.

Reuse and AddAccount (existing on-disk key, no generation) are NOT changed in
behavior — their gate keeps running against the real existing key path and they
keep working. They never persist (PersistKey is a guaranteed no-op for them).

Purpose: SAFE-03 says `--dry-run` writes nothing; the gate is a pre-WRITE gate, so a
failure must never leave an orphan key. Today `deps.Generate` writes the pair to
`~/.ssh` before the gate, violating both.
Output: refactored keygen material API, temp-then-promote orchestration in
identity, rewired buildDeps, and the five new behavioral tests.
</objective>
