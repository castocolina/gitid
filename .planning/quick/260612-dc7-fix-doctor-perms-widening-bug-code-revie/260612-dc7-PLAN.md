<objective>
Fix the doctor perms widening bug found in the Phase 4 independent code review (Important severity). `checkPath` currently flags any file whose mode is not exactly equal to the KEY-02 target and "fixes" it by chmod-ing to that target. For a private key a user hardened to 0400, this (a) falsely flags it and (b) chmods it to 0600 — ADDING owner-write, a widening that contradicts the `cmd/gitid/doctor.go` "never widens" comment and threat-model items T-04-02 / T-04-19.

The fix is surgical and tighten-only:
1. `checkPath` flags ONLY when the actual mode is LOOSER than the target: `got &^ want != 0`. When `got &^ want == 0` (equal or stricter), return no finding.
2. The fix chmods to the safe target `got & want` (clears only the offending looser bits, never sets a bit the file lacked) instead of the fixed `want`.

Purpose: Close the over-tightened edge case the review found and make the "never widens" guarantee actually enforced at the check layer (not just trusted in the FixPerm closure).
Output: tighten-only `checkPath`, a RED-first test proving the new behavior, an accurate comment in doctor.go, and updated T-04-02 / T-04-19 evidence in 04-SECURITY.md.
</objective>
