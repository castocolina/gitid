# Quick 260609-s8j — WriteFragment must ensure its parent dir exists (E2E BUG-5)

## Problem (reproduced by live E2E)

`gitid identity add` create-new flow aborts mid-write on the **third** of the four
coordinated writers:

```
Error: identity: writing gitconfig fragment:
git config --file /Users/ramon/.gitconfig.d/ramon user.name:
exit status 255: error: could not lock config file /Users/ramon/.gitconfig.d/ramon: No such file or directory
```

`internal/gitconfig/fragment.go::WriteFragment` shells out to `git config --file
<fragmentPath> …`, but `git config` does **not** create the parent directory.
When `~/.gitconfig.d/` does not yet exist (a fresh machine), every fragment write
fails. The other three writers go through `filewriter` (which ensures dirs); the
fragment writer bypasses it by shelling to git directly.

This is also a partial-write hazard: WriteSSH and WriteGitconfig (writers 1 & 2)
already mutated the user's real config before this failure. The fix makes the
fragment writer self-sufficient so the full four-artifact write completes.

## Fix

In `WriteFragment`, before the `gitConfigSet` loop, ensure the fragment's parent
directory exists via the project chokepoint:

```go
if err := filewriter.EnsureDir(filepath.Dir(fragmentPath), 0o700); err != nil {
    return fmt.Errorf("ensuring fragment dir: %w", err)
}
```

Add imports `path/filepath` and `github.com/castocolina/gitid/internal/filewriter`.
Mode `0o700` matches gitid's conservative posture for managed dirs. Do not change
`gitConfigSet`, validation, or any other writer.
