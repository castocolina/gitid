# Phase 03.1: Baseline Global Git Config + Global Gitignore - Context

**Gathered:** 2026-06-11
**Status:** Ready for planning

<domain>
## Phase Boundary

Phase 3.1 delivers gitid's **global baseline layer** — the machine-wide,
company-agnostic git configuration that sits *underneath* the per-identity
fragments built in Phases 2–3. It adds three managed surfaces, all written
through the existing safe-write chokepoint (`internal/filewriter`) with
timestamped backup → unified preview → single confirm → idempotent whole-block
rewrite, and all re-derivable from disk (no sidecar DB, consistent with IDENT-07):

- **Baseline git-config (GLOBAL-01):** a curated set of shared toggles + aliases
  in a gitid-managed block.
- **Global gitignore (GITIGNORE-01):** `core.excludesfile` → a gitid-managed
  `~/.gitignore_global` seeded with curated OS/editor/build excludes.
- **HTTPS→SSH URL rewrites (URLRW-01):** an optional curated `[url] insteadOf`
  set offered (editable) during baseline setup.

**Layering model (user's framing, recorded as the scope anchor):**
- **Baseline = "my preferences that are the same everywhere"** — pager, push/pull/
  fetch behavior, newline handling, case-sensitivity, aliases, color. Owned by
  this phase.
- **Identity fragment = "the things that vary per host/company"** — email, name,
  SSH key, signing. Owned by Phases 2–3. **Phase 3.1 does not touch them, and the
  baseline carries no `[user]` section** (this is what prevents wrong-email
  commits).

**Phase is in MVP mode** (`**Mode:** mvp` in ROADMAP.md): plans are vertical
slices over the existing `filewriter` / `gitconfig` packages — no new low-level
machinery is required, only composition of proven primitives.

**In scope (Phase 3.1 requirements):** GLOBAL-01, URLRW-01, GITIGNORE-01.

**Out of scope:**
- Per-identity fields (`[user]` name/email, SSH key, signing) — owned by Phases
  2–3 fragments; baseline never sets them.
- `core.editor` and any other strictly personal key the user did not request —
  left to `$EDITOR`/`$VISUAL` (see D-12).
- Doctor's deep checks on the baseline (e.g. detecting `ignorecase` drift, empty
  `excludesfile`, missing curated excludes) — deferred to Phase 4 (see Deferred).
- Per-host `insteadOf` suggestion wired into the identity-add flow — explicitly
  declined in favor of the baseline-time curated list (D-05).
- Full Cobra surface + TUI (Phase 5) — Phase 3.1 wires a minimal real
  `gitid baseline …` subcommand following the Phase 2/3 pattern.

<decisions>
## Implementation Decisions

### Baseline placement (architecture)
- **D-01:** The baseline lives in a **separate file**, `~/.gitconfig.d/00-baseline`,
  wired into `~/.gitconfig` via a managed `[include]` block — NOT inline in
  `~/.gitconfig`. Chosen over an in-`~/.gitconfig` managed block to match the
  user's reference gist structure (`[include] path = …`) and keep the
  `~/.gitconfig` footprint to a single managed block. The `00-` prefix groups it
  with — and orders it ahead of — the per-identity fragments already living in
  `~/.gitconfig.d/<identity>`. (User explicitly reaffirmed the `.gitconfig.d/`
  subfolder.)
- **D-02:** The baseline file is **owned via a managed block inside it**
  (`# BEGIN/END gitid managed: baseline`), not whole-file ownership. Any personal
  git settings the user adds to `~/.gitconfig.d/00-baseline` outside the block are
  preserved verbatim on re-run — consistent with the safe-write rule everywhere
  else.

### Baseline scope (what the block contains)
- **D-03 (locked by SC-1, always written):** `core.ignorecase=false`,
  `core.excludesfile=~/.gitignore_global`, `push.autoSetupRemote=true`,
  `pull.rebase=true`, `fetch.prune=true`, `color.ui=auto`. These values are fixed
  by the success criteria and were not re-debated.
  - ⚠️ Wording note: the user said "case insensitive," but `ignorecase=false`
    means case-**sensitive** filenames — the cross-platform-safe choice locked by
    SC-1 and the gist. Keep `false`.
- **D-04 (Tier 2 — seeded, editable in preview):** beyond the locked toggles, the
  baseline also seeds: `core.autocrlf=input` (the user's "newline = linux":
  commit LF, no checkout rewrite — safe on mac/linux), `core.pager=less -FRX`,
  `color.branch/diff/status=auto`, `diff.colorMoved=zebra`,
  `merge.conflictstyle=zdiff3`, `init.defaultBranch=main`, and the curated alias
  set. All Tier-2 values are shown and editable in the preview before write.
- **D-04a (aliases):** adopt the gist alias set — `st, co, br, ci, df, lg,
  unstage, last` — **editable/removable in the preview** before write (SC-1
  requires "a curated alias set").
- **D-04b (no `[user]`):** the baseline contains **no `[user]` section** and no
  per-identity field. Email/name/ssh/signing stay strictly in the identity
  fragments.

### URL rewrites — insteadOf (URLRW-01)
- **D-05 (trigger):** the HTTPS→SSH `insteadOf` rewrites are offered as a
  **curated global list at baseline setup**, NOT wired into the identity-add flow.
  This keeps URLRW-01 self-contained in Phase 3.1 and matches the gist, which
  shows a global rewrite list at the `~/.gitconfig` level.
- **D-06 (posture):** **opt-out, default-ON** for the big three — `github.com`,
  `gitlab.com`, `bitbucket.org` (→ `git@…:` `insteadOf` `https://…/`). Each
  mapping is **shown and editable/deselectable in the preview** before write, so
  the blast radius of a global rewrite (it converts *every* HTTPS URL for that
  host to SSH — can affect token/CI/read-only HTTPS clones) is surfaced and
  controllable. The edit step satisfies URLRW-01's "let the user edit it before
  generating the rewrite."
- **D-07 (block split):** the rewrites live in a **separate managed block**,
  `# gitid managed: url-rewrites`, distinct from the baseline toggle block. The
  user can have the baseline without rewrites and toggle/remove rewrites
  independently without rewriting the toggle block.

### Global gitignore (GITIGNORE-01)
- **D-08 (locked by SC-2):** `core.excludesfile` points to a gitid-managed
  `~/.gitignore_global` seeded with the curated excludes `.DS_Store`, `Thumbs.db`,
  `*.log`, `*.bak`, `*.tmp`, `*.swp` (plus curated OS/editor/build entries at
  planner discretion). Written through the safe-write chokepoint; idempotent.
- **D-09 (managed block inside the gitignore):** the curated excludes are wrapped
  in a `# BEGIN/END gitid managed: gitignore` block inside `~/.gitignore_global`,
  so the user's own ignore entries outside the block are preserved verbatim —
  parallels D-02.

### Precedence & conflict handling
- **D-10 (floor model):** the baseline is a **floor of defaults, not a ceiling**.
  The managed `[include]` block is placed near the **top** of `~/.gitconfig` so
  that the user's own hand-set keys (and the per-identity `includeIf` fragments)
  load *after* and **override** the baseline. gitid never silently overrides a
  value the user deliberately set. Identity fragments must remain ordered after
  the baseline include (natural: general defaults first, per-identity specifics
  last).
- **D-11 (detect & warn):** during setup, gitid **scans `~/.gitconfig` for keys
  the baseline also sets** and lists each conflict in the preview — the user's
  current value, the baseline value, and which one wins under the floor ordering —
  before the single confirm. Consistent with the backup→preview→confirm safety
  model; the user is never surprised by a shadowed key.

### Claude's Discretion
- **D-12 (`editor` excluded):** `core.editor` is deliberately **not seeded** — it
  is the one genuinely personal key git already solves via `$EDITOR`/`$VISUAL`.
  Left unset; the user can add it via the editable preview if desired.
- **Command surface (SC-5 read-back):** the CLI shape — e.g. `gitid baseline
  setup` / `gitid baseline show`, interactive prompts vs flags, and exactly how
  the read-back of the current managed baseline is rendered — is the planner's
  call, following the existing Phase 2/3 minimal-real-Cobra-command pattern
  (`gitid identity add/list/…`). Not throwaway; the Phase 5 surface builds on it.
- **Preview/diff rendering & confirm copy:** exact unified-preview formatting and
  conflict-warning wording — planner's call; keep it consistent with create/update.
- **gitignore curated-list extras** beyond the SC-2-locked six — planner may add
  standard OS/editor/build entries (e.g. `*.swo`, `.idea/`, `node_modules/`) at
  discretion; keep curated and conventional.
- **Package placement of new baseline read/write helpers** — keep all mutation on
  `internal/filewriter`; new baseline-specific render/read logic likely belongs in
  `internal/gitconfig` (mirrors `WriteFragment`/`WriteIncludeIf`/`ReadFragment`).
  Exact signatures are the planner's call.

## Existing Code Insights

### Reusable Assets
- **`internal/filewriter.Write`** (`filewriter.go:33`) — the single safe-write
  chokepoint (backup → temp → rename → chmod). Every Phase 3.1 mutation
  (`~/.gitconfig`, `~/.gitconfig.d/00-baseline`, `~/.gitignore_global`) routes
  through it; no `os.WriteFile` elsewhere.
- **`internal/filewriter.ReplaceBlock`** (`block.go:123`) — idempotent
  upsert-by-sentinel. The write primitive for all three managed blocks
  (`baseline`, `url-rewrites`, `gitignore`) and the `[include]` block.
- **`internal/filewriter.ListBlocks`** (`block.go:18`) + **`RemoveBlock`**
  (`block.go:59`) — enumerate/remove managed blocks; the read-back (SC-5) and
  any disable-rewrites path build on these. CRLF-tolerant (proven by tests).
- **`internal/gitconfig`** — closest analogs for the new baseline writer:
  `WriteFragment`/`RenderIncludeIf`/`WriteIncludeIf` (raw managed-text render +
  safe write), `SetAllowedSignersFile` (`fragment.go:87` — the proven "set one
  git-config key" pattern), `ReadFragment`/`ParseManagedIncludeIf` (`reader.go` —
  read-side patterns for SC-5 read-back).
- **`internal/identity` Deps pattern** (`modes.go`, `update.go`, `delete.go`) —
  every external effect is an injected function field, so logic is fake-testable
  and TUI-reusable. Baseline setup should follow the same shape.

### Established Patterns
- Core is **test-first (TDD)**; config parse → render → parse must be round-trip
  stable (TOOL-04). Idempotency (SC-1: "a second run produces no diff") is a
  natural round-trip test: write → re-write → assert no diff + foreign content
  preserved verbatim.
- Every mutation: timestamped backup before any change (SC-3), idempotent whole-
  block rewrite (never blind append), correct permissions, explicit single confirm.
- `make test` / `make lint` (golangci-lint + gosec, hard-fail) gate every commit;
  pre-push runs `go test -race` + coverage. All content English. Commits squashed
  at plan close + review.
- `gsd-tools.cjs` needs Volta's bin on PATH in non-interactive shells (GSD scripts
  only — not the Go build): `export PATH="$HOME/.volta/bin:$PATH"`.

### Integration Points
- New minimal `gitid baseline …` Cobra command in `cmd/gitid` (shape =
  planner/Phase-5 call), gathering input + building Deps from the real internal
  packages and calling a baseline operation — mirroring how `add`/`list`/`update`
  orchestrate identity ops.
- Baseline read-back (SC-5) reads `~/.gitconfig` (managed `[include]` block via
  `ListBlocks`), `~/.gitconfig.d/00-baseline` (the `baseline` block) and
  `~/.gitignore_global` (the `gitignore` block) — no sidecar DB, consistent with
  identity reconstruction.
- Conflict scan (D-11) reads existing `~/.gitconfig` keys (via `git config
  --file --list`/`--get-regexp`) and diffs against the baseline key set.

<specifics>
## Specific Ideas

- Baseline file path: `~/.gitconfig.d/00-baseline` (subfolder, `00-` orders it
  first/grouped with identity fragments) wired via a managed `[include]` block
  near the **top** of `~/.gitconfig` (floor model, D-01/D-10).
- Three managed blocks, three sentinels: `baseline` (toggles+aliases),
  `url-rewrites` (insteadOf), `gitignore` (curated excludes) — independently
  idempotent.
- insteadOf default-ON set: `github.com`, `gitlab.com`, `bitbucket.org`, each
  editable/deselectable in preview (D-06).
- Tier-2 seeded keys: `autocrlf=input`, `pager=less -FRX`, `color.branch/diff/
  status=auto`, `diff.colorMoved=zebra`, `merge.conflictstyle=zdiff3`,
  `init.defaultBranch=main`, aliases `st/co/br/ci/df/lg/unstage/last` (D-04).
- Conflict preview: for each baseline key already set outside the block, show
  `current → baseline → winner (your value, floor ordering)` before confirm (D-11).
- Idempotency proof (SC-1): write → re-run → assert zero diff + foreign content
  verbatim.

</specifics>
