## Customization & opt-in scope

### How far the baseline reaches beyond the locked SC-1 toggles

| Option | Description | Selected |
|--------|-------------|----------|
| Add safe ergonomics, skip editor/pager | Aliases + color sub-keys + diff.colorMoved + merge.conflictstyle | (recommended) |
| Locked toggles + aliases only | Minimal | |
| Full gist baseline | Includes editor=vim, autocrlf=input, pager | |

**User's choice (free-text):** *"Lo único particular que querría de config per host sería el ssh identity, email, name, signature, lo demás es opcional pero lo requerido por host es eso porque quizás para cada empresa el email es distinto y el ssh id, lo demás debería ser bastante similar."*

Reframed the question into a layering boundary: per-host = ssh identity / email / name / signing (owned by identity fragments, Phases 2–3); baseline = "everything else," shared and fairly standard. Then asked for a recommendation. Final agreed scope (confirmed via follow-up):

- **Tier 1 (locked SC-1):** ignorecase=false, excludesfile, push.autoSetupRemote, pull.rebase, fetch.prune, color.ui.
- **Tier 2 (seeded, editable):** autocrlf=input, pager=less -FRX, color.branch/diff/status, diff.colorMoved=zebra, merge.conflictstyle=zdiff3, init.defaultBranch=main, curated aliases.
- **Excluded:** core.editor (personal — left to $EDITOR); no `[user]` section (per-host only).
