# Phase 03.1 Plan 01: PrependBlockIfNotFound ‚Äî Summary

**One-liner:** `PrependBlockIfNotFound` pure `[]byte‚Üí[]byte` primitive that places a managed block at file top on first write (D-10 floor model) and delegates to `ReplaceBlock` for in-place update on re-run.

## What Was Built

Added `PrependBlockIfNotFound(existing []byte, name, blockBody string) []byte` to `internal/filewriter/block.go` (additive ‚Äî no new file) and corresponding RED/GREEN tests in `internal/filewriter/block_prepend_test.go`.

The function:
- Scans for an existing managed block using the same CRLF-tolerant `strings.TrimRight(line, "\n\r")` marker matching as `ReplaceBlock`
- If the block already exists at any position ‚Üí delegates to `ReplaceBlock` (in-place update, preserves floor position)
- If not found and `len(existing) == 0` ‚Üí returns `[]byte(block)` (empty file case)
- If not found and existing is non-empty ‚Üí `append([]byte(block), existing...)` ‚Äî prepend, placing the `[include]` at the top so git evaluates it as a floor default (RESEARCH C1 / D-10)
