<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| filesystem → dashboard/list | doctor checks + identity.Reconstruct read ~/.ssh/* and ~/.gitconfig (via tuiDeps, gitid-managed paths) |
| async tea.Cmd goroutines → model | family check results arrive asynchronously; refresh can race |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-09 | Tampering | stale async family result overwriting fresh state | mitigate | runID embedded in familyResultMsg; update() drops msg.runID != m.runID (RESEARCH Pitfall 4) |
| T-05-10 | Information Disclosure | dashboard renders doctor findings (may include paths) | accept | Paths/findings are the same already shown by the audited `gitid doctor` CLI; no new exposure |
| T-05-11 | Tampering | doctor checks shell out (ssh-add etc.) via Check* fields | mitigate | Reuses existing internal/doctor/checks arg-slice (G204-clean) exec; TUI adds no new exec call |
| T-05-12 | Denial of Service | blocking check call freezing the event loop | mitigate | Each family runs in its own tea.Cmd goroutine; nothing blocking in update() (RESEARCH Pitfall 7) |
| T-05-SC | Tampering | package installs | mitigate | No new package installs (deps from 05-01) |
</threat_model>
