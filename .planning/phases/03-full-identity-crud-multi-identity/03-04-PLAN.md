<objective>
Deliver the `gitid identity delete` vertical slice (IDENT-05): remove an identity's four per-identity managed artifacts (SSH Host block, includeIf block, fragment file, allowed_signers line) with a "will remove" manifest, a single confirm, and a backup of every mutated file (D-08). Shared/global blocks are never touched. The private key is kept by default; deleting it requires a separate explicit second prompt (default no, D-07) — block removal is reversible, key deletion is not.

After this plan a user can run `gitid identity delete <name>`, see exactly what will be removed, confirm once, and have only that identity's managed pieces removed while all foreign content (and the `Host *` / global signing wiring) stays verbatim — completing identity CRUD.

Purpose: the "D" of CRUD, with the irreversible-action safety boundary made explicit.
Output: `identity.Delete`/`DeleteDeps`/`DeleteResult` and `cmd/gitid/delete.go`.
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| CLI `<name>` arg → block removal + path build | the name selects which managed block + fragment file + allowed_signers line to remove |
| disk config → reconstruction load | account loaded from untrusted `~/.ssh/config` / `~/.gitconfig` bytes |
| key path → irreversible os.Remove | private/public key files deleted only behind the second prompt |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-14 | Tampering | wrong-identity / name-confusion deletion | mitigate | `sanitizeName`+`identityNameRe` validate the name; the "will remove" manifest is shown before the single confirm so the user sees exactly which blocks/files/line will go (D-08) |
| T-03-15 | Tampering | deleting foreign or global content | mitigate | Delete passes ONLY `acct.Name` to `RemoveBlock`; the `_global` SSH block and global `gpg.ssh.allowedSignersFile` are never referenced (D-08); RemoveBlock preserves all foreign content byte-for-byte (Success Criterion 3) |
| T-03-16 | Repudiation | irreversible key deletion with no recovery | mitigate | Key deletion is gated behind a separate explicit second prompt defaulting to "no" (D-07); block/file removals are backed up first (reversible), key deletion is intentionally not (documented) |
| T-03-17 | Information disclosure | allowed_signers rewrite / backups world-readable | mitigate | `RemoveAllowedSignersLine` writes at 0600; `filewriter` backups use 0600 (private material) |
| T-03-18 | Denial of service | deleting a non-existent or partial identity | mitigate | Reconstruct-load errors with a clear "no gitid-managed identity named" message; `RemoveBlock` on an absent block is a no-op (idempotent) — no crash, no foreign damage |
| T-03-SC | Tampering | npm/pip/cargo installs | accept | No package installs; Phase 3 adds zero external deps (RESEARCH §"No New External Packages") |
</threat_model>
