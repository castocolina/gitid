<objective>
Build the Phase 3 read-side foundation: the managed-block list/remove/backup primitives and the cross-file reconstruction join that rebuilds `[]Account` purely from the gitid-managed blocks in `~/.ssh/config` and `~/.gitconfig` (IDENT-07, D-01/D-02). No sidecar database.

This is the foundational MVP slice — `list`, `update`, and `delete` (plans 02/03/04) all key off `Reconstruct`, `ListBlocks`, `RemoveBlock`, and `BackupAndRemove`. After this plan a developer can call `identity.Reconstruct(sshBytes, gcBytes, readFrag)` and get back the exact set of identities (with incompleteness markers), proven by a round-trip test that writes via the Phase 2 pipeline and reads back.

Purpose: enable cold-start reconstruction (IDENT-07) — the precondition for every other Phase 3 capability.
Output: `ListBlocks`, `RemoveBlock`, `BackupAndRemove`, `ParseManagedHosts`, `ParseManagedIncludeIf`, `ReadFragment`, `RemoveAllowedSignersLine`, the `Account.Incomplete` field, and `Reconstruct`.
</objective>

    BackupAndRemove:
    - Existing file → creates `<path>.bak.<ts>` copy via rename, original gone, returns backupPath.
    - Missing file → returns ("", nil), idempotent (no error).

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| config files → parser | `~/.ssh/config`, `~/.gitconfig`, fragment files are read as untrusted bytes (may be hand-edited); parsers must never crash or splice foreign content |
| fragment path → git exec | fragment path passed to `git config --file <path> --list`; arg-slice exec only, never a shell |
| identity email → line filter | email used to match the allowed_signers line to remove |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-01 | Tampering | `RemoveAllowedSignersLine` over-match | mitigate | Require BOTH email AND `namespaces="git"` token to match before removing a line (Pitfall D); hand-written lines with the same email but a different namespace are preserved |
| T-03-02 | Tampering | `ReadFragment` git exec | mitigate | Arg-slice `exec.Command("git","config","--file",path,"--list")` — no shell, gosec G204-clean per established fragment.go pattern |
| T-03-03 | Information disclosure | reader error messages leaking paths/key material | mitigate | Error messages name the config-file path identifier only; never read, log, or echo private-key bytes (readers only touch config text and `.pub`/path strings) |
| T-03-04 | Denial of service | malformed managed block crashing the parser | mitigate | `ListBlocks` skips BEGIN-without-END; `ParseManagedHosts` returns zero-value SSHHostInfo on a parse error (best-effort D-02) — no panic, no abort |
| T-03-05 | Information disclosure | allowed_signers rewrite world-readable | mitigate | `RemoveAllowedSignersLine` writes via `filewriter.Write(path, result, 0o600)` (private material) |
| T-03-SC | Tampering | npm/pip/cargo installs | mitigate | No package installs in this plan (Phase 3 adds zero external deps per RESEARCH §"No New External Packages"); nothing to slopcheck |
</threat_model>
