<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| CLI `<name>` arg → block keys + exec | the identity name selects which managed block to rewrite and feeds path resolution |
| edited field values → fragment/git exec | git name/email/match values written via `git config --file` arg-slice |
| disk config → reconstruction load | current account loaded from untrusted `~/.ssh/config` / `~/.gitconfig` bytes |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-09 | Tampering | identity name in block rewrite / path build | mitigate | `sanitizeName` + `identityNameRe` (`^[A-Za-z0-9._-]+$`) reject whitespace/metacharacters before the name keys any ReplaceBlock or path (reuse rotate.go) |
| T-03-10 | Tampering | edited values → `git config --file` exec | mitigate | `WriteFragment`'s existing `validateValue`/`validateEmail` guards (newline / `[remote]` / email shape) plus arg-slice exec — no shell |
| T-03-11 | Tampering | unconfirmed mutation | mitigate | Single explicit `confirm` before any write (SAFE-03); `--dry-run` previews without writing; every mutated file backed up first by `filewriter.Write` (SAFE-01) |
| T-03-12 | Repudiation | signing-off leaving a stale allowed_signers line | mitigate | Signing-off path calls `RemoveAllowedSignersLine` (matched by email+namespace) so the verification surface reflects the new state |
| T-03-13 | Tampering | accidental rename via name edit | mitigate | `edited.Name = existing.Name` forced in `Update` (D-04) — name can never change through update |
| T-03-SC | Tampering | npm/pip/cargo installs | accept | No package installs; Phase 3 adds zero external deps (RESEARCH §"No New External Packages") |
</threat_model>
