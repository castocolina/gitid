## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| user → gitid | existing key path (reuse) / identity name (rotate/alias) entered by user |
| gitid → existing private key | reuse reads an existing private key to derive the .pub |
| gitid → four config files | rotate/alias re-point or add managed blocks (incl. allowed_signers) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-28 | Information Disclosure | derived `.pub` written world-readable / private key path mishandled | mitigate | derived .pub written 0644 via filewriter; never copy/print the private key body, only its path |
| T-02-29 | Tampering | rotation duplicates old key references instead of replacing | mitigate | idempotent `ReplaceBlock` keyed by identity name rewrites the whole managed block across all four artifacts incl. allowed_signers (SAFE-02); backup first (SAFE-01) |
| T-02-30 | Spoofing | reused key offered for wrong identity (too-many-keys) | mitigate | reuse path still emits `IdentitiesOnly yes` + explicit IdentityFile; resolved `ssh -G` asserts the right key |
| T-02-31 | Repudiation | rotation/alias write without consent | mitigate | confirmation gate retained for all modes (SAFE-03) |
| T-02-32 | Tampering/Elevation | command injection via identity name in rotate/alias args | mitigate | validate name/alias charset; arg-slice exec only (gosec G204) |
</threat_model>
