## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| gitid → `~/.gitconfig` + fragment | gitid writes includeIf headers (raw) + fragment keys (via git) |
| gitid → `ssh`/`git` binaries | tester + fragment shell out with identity-derived args |
| provider network ← `ssh -T` | tester opens a network connection to classify auth |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-18 | Tampering/Elevation | OS command injection via identity/email/alias in git/ssh args | mitigate | arg-slice `exec.Command` only (no shell); validate name/email/alias charset before interpolation (gosec G204) |
| T-02-19 | Spoofing | signing key stored inline → stale after rotation / wrong principal | mitigate | `user.signingkey` is the .pub PATH, never inline (SIGN-02) |
| T-02-20 | Denial of Service | `[remote]` in a hasconfig fragment → hard git circular error | mitigate | reject `[remote]` at render time (Pitfall 9) |
| T-02-21 | Spoofing | wrong-key auth masked by trusting `ssh -T` exit code | mitigate | classify strictly by output substring; exit code corroborating only (Pitfall 2) |
| T-02-22 | Information Disclosure | full key filesystem paths logged in user output | mitigate | Result/log surfaces alias/identifier; avoid emitting full key paths in user-facing strings |
</threat_model>
