<objective>
Make the REAL `gitid` binary open the approved app shell and drive the create
flow with real backend state. Three moves:

1. **D-14 archive** — remove the 0.0.1 POC Cobra command surface AND the old
   pre-redesign `tui/` package (per the orchestrator directive "archive the old
   app": both are superseded and cannot render the approved design). Keep
   `debug` + Cobra `completion`. Git history + `.planning/archive/` preserve them.
2. **D-15 rewire** — bare `gitid` in a TTY launches `tuikit.NewApp(realBackend)`
   rendering the approved chrome, replacing the old `tui.Run()` call site.
3. **Real Backend composition root** — a `wiring.go` that constructs the real
   `tuikit.Backend`, filling every create-flow effect (algorithm catalog,
   provider defaults, preview, alias-collision, key-scan, two-stage test,
   storage auto-detect, confirm-write persist, clipboard) from the already-built
   internal packages, and every `identity.Deps` field non-nil per the project's
   injected-seam wiring rule. This delivers SSHUI-04 (staged temp test),
   SSHUI-05 (macOS globals), and TEST-03 (store-or-adopt with backup) at the
   backend seam; the render deltas layer on in plans 03-04/03-05.

Purpose: turn the shared package into a working real application; give the
create flow a real spine end-to-end for the states already in the approved design.
Output: archived POC/tui, rewired main, the real Backend composition root.
</objective>
