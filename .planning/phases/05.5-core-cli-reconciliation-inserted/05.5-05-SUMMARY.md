# Phase 05.5 Plan 05: Shared Overlap/Ambiguity Detector Summary

Shared overlap detector (`DetectOverlaps` + `CheckOverlap`) with three detection
kinds (identical-gitdir, nested-gitdir, hasconfig-url), wired into `gitid doctor`
as `FamilyOverlap` (warning tier) and into `add`/`update` as a write-time
warn+confirm gate (D-16).

## What Was Built

### Task 1 (TDD RED → GREEN): `internal/doctor/checks/overlap.go`

**`OverlapPair`** struct: `A`, `B` (identity names), `Kind` (overlap type),
`Detail` (human-readable description).

**`DetectOverlaps(accounts []identity.Account) []OverlapPair`:**
- Filters reserved blocks (`gitconfig.IsReservedBlockName`) and `_global` accounts
  before any comparison (Pitfall 4 / T-05.5-15).
- Pairwise O(n^2) scan comparing all match combinations for each unordered pair.
- Three overlap kinds detected (D-14):
  1. `identical-gitdir`: both accounts have the same gitdir after trailing-slash
     normalization (`normGitdir`).
  2. `nested-gitdir`: one gitdir is a prefix of the other (e.g. `~/git/` vs
     `~/git/personal/`).
  3. `hasconfig-url`: conservative heuristic — equal patterns, one a prefix of the
     other, or both share the same `git@<host>:` prefix.

**`CheckOverlap(deps doctor.Deps) []doctor.Finding`:**
- Wraps each `OverlapPair` as `doctor.Finding{Family: FamilyOverlap, Severity: SeverityWarning}`.
- `Fix: nil` — overlaps are advisory, not auto-fixable (T-05.5-15).
- `Explanation` names both identities and notes the last-written-wins semantics.

**`doctor.FamilyOverlap Family = "Overlap"` constant** added to `internal/doctor/doctor.go`.

**`cmd/gitid/doctor.go` (`buildDoctorDeps`):**
- `CheckOverlap: checks.CheckOverlap` wired alongside the other 7 check functions.
- `Identities` was already populated in the deps — no additional setup needed.

**`cmd/gitid/add.go` — new helpers:**
- `warnOverlapAndConfirm(r, out, prospective, existing)`: calls `DetectOverlaps`,
  filters to pairs involving the prospective identity, prints warning with kind+detail
  for each overlap, and returns `confirm(r, out, "Proceed with overlapping match conditions?")`.
  Returns `true` immediately when no overlaps detected (common path, no prompt).
- `loadExistingAccounts()`: reads `~/.ssh/config` and `~/.gitconfig` via
  `identity.Reconstruct`; returns empty slice on error (non-fatal; new installs have
  no existing accounts to compare against).
- `warnOverlapAndConfirm` called in `runCreateNew` after the auth-gated loop passes
  but BEFORE `identity.PersistAll` (D-16).

**`cmd/gitid/update.go`:**
- `warnOverlapAndConfirm` called before `confirm(... "Apply these changes now?")`.
- "Others" list = all reconstructed accounts except the one being updated.
