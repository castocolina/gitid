## What Was Built

### internal/platform/install.go (new)

`BinaryInstallInfo() (path string, onPATH bool, err error)`:
- `os.Executable()` to get the running binary path
- `filepath.EvalSymlinks(exe)` to resolve symlinks (tolerates error per A1, keeps unresolved path on failure)
- Delegates PATH check to `binaryOnPath(exePath, os.Getenv("PATH"))`

`binaryOnPath(exePath, pathEnv string) bool`:
- Pure helper (no os.Getenv): splits `pathEnv` on `os.PathListSeparator`, compares `filepath.Dir(exePath)` to each entry
- Unit-testable without the real running binary (tests inject a controlled PATH string via table tests)

### cmd/gitid/doctor.go (modified)

`installInfoFinding() doctor.Finding`:
- Calls `platform.BinaryInstallInfo()`
- Returns `doctor.Finding{Family: doctor.FamilyDeps, Severity: doctor.SeverityInfo}`
- Title: `"binary: <resolved-path>"`
- Explanation: `"on PATH: yes"` or `"on PATH: no"`
- SuggestedFix (only when off-PATH): `export PATH="$PATH:<binDir>"`
- `Fix: nil` — info-only, never auto-fixable, never raises exit tier

`runDoctor` change: appends `installInfoFinding()` to `findings` after `doctor.Run()` and before `renderReport()`.

### Makefile (modified)

`install` target now echoes:
```
  installed: /Users/ramon/go/bin/gitid
  PATH: OK (gitid is on PATH)
```
or:
```
  installed: /Users/ramon/go/bin/gitid
  PATH: /Users/ramon/go/bin is NOT on your PATH — add to shell: export PATH="$PATH:/Users/ramon/go/bin"
```

Uses `$(GOPATH_BIN)` (already defined at line 22) — no new variables.
