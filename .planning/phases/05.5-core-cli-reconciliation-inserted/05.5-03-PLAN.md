artifacts now?" prompt (D-02) and the temp-stage→delete behavior.

un-authenticated key. Output: a trustworthy create-flow with a CI-testable loop gate (the E2E
shells from 05.5-01 turn green in 05.5-07).

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| user stdin → loop choice parsing | retry/skip/quit selection drives whether config is persisted |
| generated private key → ~/.ssh | key material written to disk before any auth (D-01) |
| ssh connectivity output → PASS classifier | provider banner determines the persist gate |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05.5-07 | Spoofing | treating reachability as authentication | mitigate | Gate strictly on `tester.PASS` (successfully-authenticated banner); `ReachableNotUploaded` (Permission denied) explicitly does NOT pass (D-06) — guarded by a dedicated regression test |
| T-05.5-08 | Elevation | private key perms on early write | mitigate | Generate writes via `filewriter.Write(path, pem, 0o600)` / `.pub` 0o644 through the chokepoint (KEY-02); never `os.WriteFile` directly (CLAUDE.md) |
| T-05.5-09 | Tampering | skip-&-write bypasses the auth gate silently | mitigate | `[s]` requires an explicit typed `confirm` after a "not yet authenticated" warning (D-05); never auto-approved |
| T-05.5-10 | Information Disclosure | orphan key left after quit | accept | Key stays 0600 in ~/.ssh; doctor orphan check flags it (D-04 intended); the loop prints its path |
| T-05.5-11 | Tampering | private key bytes logged/printed | mitigate | PrivPEM never printed; only `.pub` line copied/shown (existing StagedKey contract preserved) |
| T-05.5-SC | Tampering | npm/pip/cargo installs | mitigate | No package installs; existing deps only |
</threat_model>
