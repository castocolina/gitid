<objective>
Author the cross-OS GitHub Actions CI (BUILD-01/02/04) that gates the whole phase, restructured per Codex review for cost, supply-chain, and least-privilege:

- `build-cross` runs ONCE on Linux (cross-compilation via GOOS/GOARCH is OS-independent), not redundantly on every runner (Codex MEDIUM).
- A FAST PR gate runs `make setup-env` → `make test` (-race) → `make lint` on all three CORRECTED runners (`ubuntu-latest`, `macos-15-intel`, `macos-15`) on every pull_request; the heavier `make test-e2e` runs on `ubuntu-latest` + `macos-15` on PRs and on ALL THREE on push to main. This applies D-13's documented cost lever (gate the third-runner e2e to push/main) rather than D-13's full-on-PR default — a deliberate choice recorded here to address the external reviewer's cost/flakiness concern while keeping cross-OS test+lint on every PR (BUILD-02's PLAT-02 divergence coverage).
- All actions are pinned by 40-char commit SHA (tags are mutable — Codex MEDIUM), the workflow declares a least-privilege `permissions: { contents: read }` block, and NO secrets are used in Phase 1.

CI calls the SAME make targets a human runs locally (single source of truth) and reproduces the toolchain from a fresh clone via `make setup-env`. A real Actions run must be green on all three runners.

Purpose: BUILD-02 exists to catch the PLAT-02 macOS-vs-Linux divergences (keychain vs ssh-agent, LibreSSL vs OpenSSL, clipboard) built earlier this phase. CRITICAL: use the macos-15 labels — `macos-13` is unsupported since Dec 2025 and `macos-14` begins deprecating 2026-07-06 (RESEARCH State of the Art; Codex independently confirmed the macos-15 labels are current). Release/tag publishing (BUILD-03) is Phase 10, OUT OF SCOPE.
Output: `.github/workflows/ci.yml` + a `build-cross` Makefile target.
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| CI runner ← repo code + third-party actions/installers | hosted runners execute the repo's make targets and pinned marketplace actions |
| CI ← setup-env installers (golangci-lint, gosec, freeze, pre-commit, pinned Chromium) | supply-chain surface reproduced on every run |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-SC3 | Tampering | pinned dev-tool installers in CI setup-env | mitigate | Versions pinned (golangci-lint v2.12.2, freeze @v0.2.2, go-rod @v0.116.2, explicit Chromium revision); CI reuses the SAME pinned make targets — no `@latest` drift on the gated path |
| T-01-18 | Tampering | GitHub Actions marketplace actions | mitigate | Pin checkout/setup-go/cache actions to full 40-char commit SHAs (tags are mutable); acceptance greps assert SHA pinning and reject `@vN` tag-only refs |
| T-01-24 | Elevation of Privilege | over-broad workflow token | mitigate | Declare a least-privilege `permissions: { contents: read }` block; no `secrets.` references anywhere in Phase 1 |
| T-01-19 | Elevation of Privilege | untrusted PR code running in CI | accept | Solo-developer local-use tool; standard GitHub-hosted-runner isolation; read-only token; no secrets required for the Phase-1 gates (release/publish with secrets is Phase 10) |
| T-01-20 | Denial of Service | stale runner labels (macos-13/14) block scheduling | mitigate | Use the corrected macos-15-intel/macos-15 labels; acceptance greps assert macos-13/macos-14 are absent |
</threat_model>
