<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| CI runner ← repo code + third-party actions/installers | hosted runners execute the repo's make targets and pinned marketplace actions |
| CI ← setup-env installers (golangci-lint, gosec, freeze, pre-commit, Chromium) | supply-chain surface reproduced on every run |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-SC3 | Tampering | pinned dev-tool installers in CI setup-env | mitigate | Versions pinned (golangci-lint v2.12.2, freeze @v0.2.2, go-rod @v0.116.2, Chromium revision); CI reuses the SAME pinned make targets — no `@latest` drift on the gated path |
| T-01-18 | Tampering | GitHub Actions marketplace actions | mitigate | Pin checkout/setup-go/cache actions to explicit version tags rather than a floating major |
| T-01-19 | Elevation of Privilege | untrusted PR code running in CI | accept | Solo-developer local-use tool; standard GitHub-hosted-runner isolation; no secrets required for the Phase-1 gates (release/publish with secrets is Phase 10) |
| T-01-20 | Denial of Service | stale runner labels (macos-13/14) block scheduling | mitigate | Use the corrected macos-15-intel/macos-15 labels; acceptance greps assert macos-13/macos-14 are absent |
</threat_model>
