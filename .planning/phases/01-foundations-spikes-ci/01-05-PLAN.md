<objective>
Build the repeatable screenshot tooling (TOOL-05, DLV-03): a `make screenshot-tui` target that renders a deterministic Bubble Tea `View()`-dump golden to a PNG via `charmbracelet/freeze` (pinned, with a vendored monospace font for CI determinism), and a `make screenshot-html` target that renders a fixture HTML page to a PNG via a headless-Chromium `go-rod` driver. Both write versioned PNGs under `.planning/design/<surface>/{tui,html}/`. The two new dependencies are dev/build-tools only — isolated behind a `//go:build screenshot` tag so `go build ./cmd/gitid` never pulls a headless browser into the shipped binary. `make setup-env` installs `freeze@v0.2.2` and provisions the browser (extends TOOL-02).

Purpose: Phase 2 (design) and every later UI wave call these targets to capture reference/regression screenshots. Phase 1 builds the tooling and proves it against a trivial fixture (mockups do not exist until Phase 2). NOTE: not the Playwright MCP (agent-only; cannot run in make/CI).
Output: `Makefile` targets, `internal/screenshot/{tui,html}.go`, a vendored font, and a fixture HTML page.
</objective>
