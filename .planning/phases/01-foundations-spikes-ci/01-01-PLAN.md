<objective>
Extend `internal/platform` with the local-capability probe layer that PLAT-01/PLAT-02/KEY-03 depend on: parse `ssh -V` into a structured `SSHVersion{OpenSSHVersion, SSLFlavor, SSLVersion, Raw}` value, map every `ssh -Q key` token to a catalog algorithm name (including the FIDO2 `sk-` variants that the requirements docs mis-name), and detect libfido2/ssh-sk-helper, a running ssh-agent, and macOS keychain support behind an injectable seam. Per Codex review: the probe surface returns STRUCTURED values (a `SSHVersion` struct, three-valued `AgentStatus`/`FIDOStatus`/`KeychainStatus` statuses) rather than lossy `string`/`bool`, because the doctor/fixer copy in later phases needs to distinguish "no agent" from "agent locked" and "libfido2 present but middleware unusable" from "absent". Every external probe runs under an `exec.CommandContext` timeout (`ssh-add -l` can hang on a stuck agent). Extend `InstallHint` with a libfido2 tool family for KEY-03 troubleshooting.

Purpose: The algorithm catalog (01-02) resolves per-OS availability from these probe results, and the debug/list command (01-06) reports them. This closes the recurring "injected-seam wiring blindspot" by making every probe mockable AND exercising the real wiring in tests.
Output: `internal/platform/version.go`, `capabilities.go`, `keytypes.go` (+ tests), a timeout retrofit of the existing `ProbeKeyTypes` in `platform.go`, and an extended `install.go`.
</objective>
