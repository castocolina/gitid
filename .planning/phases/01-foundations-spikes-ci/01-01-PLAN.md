<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| gitid → local OS toolchain (`ssh`, `ssh-keygen`, `ssh-add`) | probe shells out to trusted system binaries; their output crosses back as untrusted text to parse |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-01 | Tampering | capabilities.go / version.go exec calls | mitigate | Arg-slice `exec.Command` form only, never shell string interpolation; every call annotated `#nosec G204 -- fixed args, no user input`, mirroring `ProbeKeyTypes` |
| T-01-02 | Spoofing | `ssh -Q key` / `ssh -V` output parsing | mitigate | Parse into a fixed token map; unknown tokens map to empty (ignored), so a manipulated PATH `ssh` cannot inject a false catalog entry into downstream logic |
| T-01-03 | Denial of Service | probe on a machine without libfido2/agent | accept | Absence is the expected common case (RESEARCH Environment Availability); probe returns false + nil error, never blocks |
</threat_model>
