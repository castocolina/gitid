<objective>
Add the MGR-02 identity state-taxonomy core to `internal/identity`, restructured per Codex review into TWO layers so distinct facts are never collapsed and 01-06 has a real aggregation layer to consume:

1. A PURE classifier (`state.go`): keep the 8 locked MGR-02 labels as the shared `State` vocabulary, but return an `IdentityHealth{IdentityState, KeyState, Problems}` report with orthogonal identity- and key-state axes (so `fragment-path-missing` AND `key-missing` can both be reported). Retain a single-label `ClassifyState` with documented precedence for callers that want one state per identity. Includes the `crossReferenceUnusedKeys` helper.
2. A state-inventory BUILDER (`inventory.go`): the impure aggregation layer (Codex HIGH #4) that parses managed SSH blocks + git includes/fragments, stats keys/fragments, computes key usage in SSH and signing, detects unused keys, then calls the pure classifier — all behind an injectable seam. This is what the debug/list command (01-06) consumes so it never rebuilds identity logic in `cmd/gitid`.

Computed entirely from parsed managed blocks with no sidecar DB (DLV-07 UI-free TDD core).

Purpose: The identity manager (Phase 5, MGR-01/03/07) renders per-row state and the debug/list command (01-06) surfaces it in Phase 1 by consuming BuildInventory. This plan builds both the classifier and its real inventory feeder; it does NOT modify the doctor (that dedup is a Phase 8 concern — avoids a Wave-1 file conflict with 01-03's orphans.go change).
Output: `internal/identity/state.go` + `state_test.go`, `internal/identity/inventory.go` + `inventory_test.go`.
</objective>

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| parsed managed blocks → state classification | untrusted config content is parsed upstream (sshconfig/gitconfig); the pure classifier only reads the already-parsed Account shape |
| filesystem (keys/fragments/config) → inventory builder | the builder reads real paths behind an injectable stat/read seam; it never writes |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-11 | Tampering | Classify/ClassifyState inputs | accept | Read-only pure functions; no writes, no exec, no sidecar DB — a malformed Account yields a report, never a mutation. Low risk for a local single-user tool |
| T-01-12 | Information Disclosure | state/inventory fixtures | mitigate | Fixtures use synthetic identity names/paths under `t.TempDir()`; no real key material is embedded in test data; the inventory builder reads but never prints key bytes |
| T-01-23 | Tampering | inventory builder filesystem reads | mitigate | All effects (reads/stat/glob) are injected function fields; the builder performs no writes; the real constructor is test-exercised so a broken seam cannot pass on green fakes |
</threat_model>
