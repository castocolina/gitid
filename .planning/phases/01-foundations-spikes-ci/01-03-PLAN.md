| Boundary | Description |

| gitid → `~/.ssh/config` + `~/.ssh/config.d/*.config` | every mutation crosses the filewriter safe-write chokepoint; a migration touches TWO files that must stay coherent across the operation |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-07 | Tampering | adopt.go external Include path | mitigate | Selection rules: adopt only a sentinel-bearing or caller-chosen, absolute/`~/.ssh`-relative, non-symlink (`os.Lstat`), unambiguous target; reject broad globs and bare-relative paths at the boundary |
| T-01-08 | Tampering | include.go / migrate.go writes | mitigate | Route every write through `filewriter.Write` (backup + atomic temp→rename→chmod; dir 0700 / file 0600); never `os.WriteFile`; re-parse composed config before writing to reject a corrupt result |
| T-01-22 | Tampering (data integrity) | cross-file migration crash between the two writes | mitigate | Add-to-destination-before-remove-from-source ordering (worst case = recoverable duplicate, never loss) + timestamped backups of BOTH files + per-step parse/`ssh -G` validation + rollback-from-backup on failure + idempotent re-run; injected-failure tests after each write step assert no corruption |
| T-01-09 | Denial of Service | doctor Orphans false-positive delete loop | mitigate | Add `sshconfig.IsReservedBlockName` guard + Class 1 skip in the SAME change so the ssh-include block is never flagged/deleted/recreated (documented recurring bug class) |
| T-01-10 | Information Disclosure | test fixtures leaking real `~/.ssh` | mitigate | All Include-parsing tests use `t.TempDir()` + a hermetic HOME (Pitfall 5), never the runner's real home |
