<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| user-influenced Include/adopt path → filesystem write | an existing config's Include path or a user-confirmed external path becomes a migration/write target |
| gitid → `~/.ssh/config` + `~/.ssh/config.d/*.config` | every mutation crosses the filewriter safe-write chokepoint |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-07 | Tampering | adopt.go external Include path | mitigate | Validate the path at the boundary: reject non-absolute / non-`~/.ssh`-relative paths (STORE-01 locked constraint) and reject symlinks via `os.Lstat` guard before adoption (mirrors adopter.go) |
| T-01-08 | Tampering | include.go / migrate.go writes | mitigate | Route every write through `filewriter.Write` (backup + atomic temp→rename→chmod, mode 0600); never `os.WriteFile`; re-parse composed config before writing to reject a corrupt result |
| T-01-09 | Denial of Service | doctor Orphans false-positive delete loop | mitigate | Add `sshconfig.IsReservedBlockName` guard + Class 1 skip in the SAME change so the ssh-include block is never flagged/deleted/recreated (documented recurring bug class) |
| T-01-10 | Information Disclosure | test fixtures leaking real `~/.ssh` | mitigate | All Include-parsing tests use `t.TempDir()` + a hermetic HOME (Pitfall 5), never the runner's real home |
</threat_model>
