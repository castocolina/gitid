<objective>
The auto-fix slice (DOC-06 second half): wire the cmd-layer fixer that turns the report's `[fix]`
markers into confirmed, safe mutations. Implements the full D-04 consent flow — top-level gate (bare
`doctor`), per-finding confirm (`--fix`), non-interactive apply (`--fix --yes`) — with permission
batching and individual confirms for orphaned-block removal / wiring re-add. Every write routes through
internal/filewriter (backup + atomic + idempotent); the doctor core stays write-free (D-01).

Purpose: This is the only plan that mutates user files, so it is isolated in its own wave after all
read-only families exist. It depends on every prior plan because the fixer must handle the three
auto-fixable classes those plans produce: permission chmod (Plan 01), orphaned-block removal +
wiring re-add (Plans 02/03), with agent/dep/key findings report-only.

Output: extended cmd/gitid/doctor.go (gate + fixer + buildDoctorDeps fix-field wiring), cmd/gitid/doctor_test.go.
</objective>
