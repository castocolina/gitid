<objective>
Close DOC-GAP-02 and DOC-GAP-03, plus the coupled review items IN-03 and WR-03.

DOC-GAP-02 (critical, DOC-05): `buildDoctorDeps` never sets `Deps.RunSSHAdd` (or
`RunSSHKeygenFingerprint`), so `CheckAgent` always hits its
`if deps.RunSSHAdd == nil { return nil }` guard and the Agent section
unconditionally reports healthy — masking a down ssh-agent or unloaded keys.

DOC-GAP-03 (warning, DOC-06): `runDoctor` calls `applyFixes` on real `os.Stdin`
with no terminal check, so a piped/CI bare `gitid doctor` prints an unanswerable
`Apply N fix(es)?` prompt into machine-parsed output.

IN-03: `main()` collapses the tiered 0/1/2/3 `doctor.ExitCode` to a flat
`os.Exit(1)`, defeating the D-07 tiering at the process boundary.

WR-03: the permissions check flags a default 0644 `~/.gitconfig` as a
SeverityError, a noisy false positive on virtually every machine.

Purpose: make the Agent family actually run in production, keep non-interactive
doctor output clean, preserve the D-07 tiered exit code through the process
boundary, and stop a false-positive perms error.
Output: real ssh-add/ssh-keygen runners wired into Deps, a TTY-guarded fix gate,
exit-code propagation through main, a corrected gitconfig perms rule, and tests
that drive the real wiring.
</objective>
