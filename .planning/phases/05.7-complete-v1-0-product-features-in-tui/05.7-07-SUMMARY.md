# Phase 05.7 Plan 07: Surface Adopt Modal, Add Repo Modal, Upload-Assist in TUI

**One-liner:** Fragment discriminator + Adopt modal (migrate/reference state machine) + Add Repo clone flow (URL→detect→picker→rewrite→clone+pull→inline-create resume) + gh/glab upload-assist on Copy modal, all wired into the real TUI model via injected Deps seams with zero os/exec in modal files.

## What Was Built

### Task 1 — Sidebar Fragment Discriminator + Adopt Modal + Upload-Assist

**tui/sidebar.go** — `unmanagedEntry` gains a `kind` discriminator (`kindOrphanKey` / `kindFragment`) and a `fragmentPath` field. `buildUnmanaged` now populates `kindFragment` rows from `deps.adopt.ListCandidates` in Step 1 (BEFORE any [Adopt] affordance dispatch observes them — critical ordering per REVIEWS.md #2). Step 2 remains the existing orphan-key loop. `renderUnmanagedRow` shows `[Adopt]` (StyleAccent) on focused `kindFragment` rows; orphan-key rows keep the read-only `~ glyph`. Nil `ListCandidates` seam is guarded (test mode).

**tui/styles.go** — Adds `StyleAccent` (bold blue, ANSI color 4) for the `[Adopt]` affordance and the Add Repo spinner.

**tui/adopt.go** — `adoptModel` 5-phase state machine:
- `adoptPhaseConfirm`: method selector (m/r/tab toggle), Enter dispatches `runAdoptCmd`
- `adoptPhaseRunning`: spinner while `adopter.Adopt` runs via `deps.adopt` seam
- `adoptPhaseDone`: "Adopt complete" shown; Enter (Migrate) → `adoptPhaseOfferRemove`
- `adoptPhaseOfferRemove`: y/N prompt, N default, only y+Enter calls `runRemoveOriginalCmd` (D-05: never auto-removes)
- `adoptPhaseError`: error text, Esc → `adoptCancelMsg`

Default method = `AdoptMigrate` (D-04). All effects via `deps.adopt` seam — NO `os/exec`.

**tui/copy.go** — `copyPubkeyModel` extended with upload-assist (AUTOUP-01):
- Fields: `uploadDetected`, `uploadTool`, `uploadToolPath`, `uploadAuthStatus`, `uploadPending`, `uploadResults`
- `init()` runs `uploader.Detect` synchronously; pre-builds `uploadPending` queue when `AuthAuthenticated`
- `view()` renders: authenticated → per-key prompts with `CommandPreview` (shown == run); not-logged-in → notice with `gh auth login` hint; tool-not-found → section omitted entirely (D-11 silent-skip)
- `runUploadKeyCmd` derives `.pub` path from `privKeyPath + ".pub"` — ONLY the public key path is passed to `uploader.UploadKey` (T-05.7-07-04)
- `'s'` skips current prompt; Esc always closes (non-blocking per D-11)

**Tests added (Task 1):** 15 adopt tests + 4 upload tests = 19 tests.

### Task 2 — Add Repo Modal + Model Wiring

**tui/addrepo.go** — `addRepoModel` 9-step state machine:
- `addRepoStepDetect`: URL textinput; Tab/Enter dispatches `runDetectCmd`
- `addRepoStepClientPicker`: radio picker over `clientCandidates`; Enter → rewrite preview
- `addRepoStepInlineCreate`: sub-modal: `createWizardModel` inline (D-08: no-match → continuous, no abort); on wizard success restores `savedRawURL`/`savedProvider` and resumes at client picker (modal-stack invariant: `createWizard` is nil in all other steps)
- `addRepoStepRewritePreview`: shows original URL → rewritten alias URL → dest path → git clone command preview; Enter → clone
- `addRepoStepCloning` / `addRepoStepPulling`: streaming via `deps.repoclone.Clone` / `.Pull`
- `addRepoStepDone` / `addRepoStepError` / `addRepoStepDestExists`: terminal states

All clone/pull effects via `deps.repoclone` seam — NO `os/exec`.

**tui/model.go** — Full modal wiring:
- `adoptModal` and `addRepoModal` added to `modalKind` enum
- `adoptM adoptModel` and `addRepoM addRepoModel` fields added to `rootModel`
- Message routing: `adoptResultMsg`, `adoptCancelMsg`, `removeOriginalResultMsg`, `uploadKeyResultMsg`, `detectResultMsg`, `cloneResultMsg`, `pullResultMsg`
- Key routing: `'A'` → adopt modal (kindFragment only), `ctrl+r` → add repo modal
- `renderContent()`: adds `adoptModal` and `addRepoModal` rendering branches
- Footer: `ctrl+r repo` hint always shown in Identities view; `A adopt` hint shown only when a `kindFragment` entry is focused
- `applyPaletteAction`: `action:addrepo` and `action:adopt` dispatchers

**tui/palette.go** — Adds `paletteActionItems` (Add Repo + Adopt Fragment) merged into `newPaletteModel`'s item list.

**tui/keymap.go** — Adds `Adopt` (A) and `AddRepo` (ctrl+r) bindings.

**Tests added (Task 2):** 12 add-repo tests.

## Deviations from Plan

### CLAUDE.md-Driven Deviation: Unified Commit (Buildable Boundary)

**Found during:** Commit attempt for Task 1
**Issue:** `adopt_test.go` references `adoptModal` and `addrepo_test.go` references `addRepoModal`, both defined in `model.go`. When the pre-commit hook stashed unstaged `model.go`/`palette.go`/`addrepo.go`/`addrepo_test.go`, the lint+typecheck run failed with "undefined: adoptModal".
**Fix:** Combined both tasks into one commit per CLAUDE.md: "Changes that do not compile in isolation must be committed together — let the buildable boundary, not file count, set the commit granularity."
**Impact:** Both tasks share commit `a3ed2cf`.

### Auto-fix: adoptPhaseDone Before adoptPhaseOfferRemove

**Rule 1 (Bug Fix)**
**Found during:** Task 1 GREEN phase — TestAdoptModelResultMsgSuccess failed: expected `adoptPhaseDone` (phase 2), got `adoptPhaseOfferRemove` (phase 3).
**Issue:** Original `adoptResultMsg` handler immediately jumped to `adoptPhaseOfferRemove` for Migrate, skipping `adoptPhaseDone`. Test asserts adoptResultMsg{err:nil} must set adoptPhaseDone.
**Fix:** `adoptResultMsg` always sets `adoptPhaseDone`; Enter in the `adoptPhaseDone` handler then transitions to `adoptPhaseOfferRemove` for Migrate (better UX too — user sees "Adopt complete" before remove prompt).

### Auto-fix: Remove Unused Fields and Types (Rule 3 — Lint Compliance)

**Found during:** `make lint` run before commit.
- `openAdoptMsg` struct removed from `adopt.go` — declared for future routing but model.go now uses `adoptModal` kind directly without the message type.
- `err error` field removed from `adoptModel` struct — never set; only `errText string` is used.
- `makeFakeAddRepoDeps` and `fakeStatNotFound` removed from `addrepo_test.go` — both declared but never called.
- `uploadKeyResultMsg` → `uploadKeyResult(msg)` direct conversion (S1016: identical field types allow type conversion).
- `runDetectCmd` `deps` param renamed to `_` (revive: unused parameter).

## Known Stubs

`runDetectCmd` in `addrepo.go` returns `detectResultMsg{provider: provider}` without identity alias matching (the `matchedAlias` and `matchedName` fields are always empty from the live detect path). Identity matching requires sidebar accounts accessible in the cmd closure — deferred to a future plan. The client-picker UI is functional; it will show the detected provider but the matched-alias rewrite will not trigger automatically from a live URL. Tests inject the full `detectResultMsg` directly so behavior is fully tested.
