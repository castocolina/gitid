# Phase 05.7 — Validation Strategy

> Per-phase validation contract for feedback sampling during execution.
> Two layers (see 05.7-RESEARCH.md § Validation Architecture): **unit-TDD** on new
> core logic (RED/GREEN/REFACTOR), and **e2e** over the REAL program entry
> (`gitid` binary + TUI over PTY) — the latter is mandatory to close the recurring
> D-16/D-13 injected-seam blindspot.

---

## Test Infrastructure

| Property | Value |
|----------|-------|
| **Framework** | `go test` (stdlib) + `charmbracelet/x/exp/teatest/v2` (optional, TUI e2e) |
| **Config file** | none — `Makefile` targets; e2e harness in `e2e/harness_test.go` (`SandboxHome`, `BuildBinary`, `FakeSSHDir`) |
| **Quick run command** | `make test` (unit + race) |
| **Full suite command** | `make test && go test ./e2e/...` (builds binary, sandboxed HOME) |
| **Estimated runtime** | ~30–90 seconds (e2e binary build dominates) |

---

## Sampling Rate

- **After every task commit:** Run `make test` (package-scoped where possible)
- **After every plan wave:** Run `make test && go test ./e2e/...`
- **Before `/gsd-verify-work`:** Full suite (unit + e2e) must be green
- **Max feedback latency:** ~90 seconds
