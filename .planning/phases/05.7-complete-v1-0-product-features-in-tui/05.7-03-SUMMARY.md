# Phase 05.7 Plan 03: internal/repoclone — URL parsing, clone/pull seam Summary

Green `internal/repoclone` package replacing Plan 01 RED stubs: pure URL parsing (ProviderFromURL, RewriteToAlias, DestPath) + injected-seam Clone/Pull with dest-exists and dest-outside-base guards before any exec.

## What Was Built

### Task 1: ProviderFromURL + RewriteToAlias + DestPath (pure logic)

All three functions are pure (no exec, no filesystem access) and table-tested:

- `ProviderFromURL(rawURL)`: handles three URL forms — `https://`, `git@host:` SCP-like (no scheme), `ssh://`. Returns `("", ErrUnknownProvider)` for garbage input.
- `RewriteToAlias(rawURL, alias)`: extracts `org/repo` path, strips scheme+host, produces recipe-canonical `git@<alias>:<org>/<repo>[.git]`. Preserves `.git` suffix if present; never emits a stray scheme.
- `DestPath(baseDir, client, rawURL)`: joins `baseDir/client/reponame` where reponame = last path segment minus `.git`. Pure, no I/O.

### Task 2: Clone/Pull seam — guards + exec

- `Clone(cloneURL, destPath, deps)`:
  1. Derives allowed base internally: `deps.UserHomeDir() + "/git"` (no base-dir param per REVIEWS.md #10 lock).
  2. **ErrDestOutsideBase guard** — rejects `destPath` not under base via `filepath.Rel` + `..` prefix check; fires BEFORE `Stat` or exec (counter-fake test asserts `Stat` not called).
  3. **ErrDestExists guard** — rejects if `deps.Stat(destPath)` returns no error; fires before clone exec.
  4. Delegates to `deps.Clone(cloneURL, destPath)` on success.
- `Pull(destPath, deps)`: delegates directly to `deps.Pull(destPath)`.
- `liveClone`/`livePull`: package-level functions using `exec.Command("git", "clone", url, dest)` and `exec.Command("git", "-C", dest, "pull")` arg slices; annotated `//nolint:gosec // arg-slice; no shell; URL/dest validated (G204)`.
- `LiveClone`/`LivePull` exported vars wire `liveClone`/`livePull` for `tui/deps.go` and CLI.
