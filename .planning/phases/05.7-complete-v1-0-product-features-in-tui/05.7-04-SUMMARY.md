## What Was Built

The `internal/uploader` package (AUTOUP-01 core) was completed from its Plan 01 RED stub to a fully GREEN implementation. This package is the thin, testable wrapper that the copy modal and CLI `--upload-keys` flag (Plan 06/07) will compose.

### Detect

Scans PATH for `gh` then `glab` (gh always preferred â€” deterministic order). On first match, probes authentication via `<tool> auth status` (exit 0 = authenticated). Returns three states: `AuthAuthenticated`, `AuthNotLoggedIn`, `AuthToolNotFound`. No real subprocess in unit tests â€” injected via `Deps.LookPath` and `Deps.RunCmd`.

### AuthCheck

Reusable auth probe extracted as a standalone exported function for callers that only need auth status after detection.

### UploadKey

Builds an explicit arg slice per tool:
- **gh:** `ssh-key add <pub> --title <title> --type authentication|signing`
- **glab:** `ssh-key add <pub> -t <title> --usage-type auth`

Never uses `sh -c`. On non-zero exit, returns captured output plus a wrapped error so callers can display a manual fallback (D-11 / D-12). Only `.pub` paths flow through â€” no private key ever accepted.

### CommandPreview

Returns the identical arg slice as `UploadKey` joined into a display string (`toolPath + " " + args`). The shown command equals the run command (UI-SPEC Â§4a invariant), asserted by `TestCommandPreview_GHEqualsRunCmd`.
