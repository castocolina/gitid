<objective>
Run the phase-end review gates (operator requirement), gated on all four features
being built (Plans 05/06/07): (1) an AUTONOMOUS PTY-driven UI e2e that drives the
REAL binary over a pseudo-terminal and asserts on decoded frames, (2) an
agent-ui-ux-designer design critique of the four new TUI surfaces against
05.7-UI-SPEC.md (fed the real PTY frames), and (3) a requesting-code-review
(superpowers skill) pass over the complete Phase 5.7 diff. The gates feed a fix loop;
the phase does not advance to verification until the PTY e2e is green and CRITICAL/HIGH
(UI) and blocking (code-review) findings are resolved or deferred with written
rationale. A real-SSH-terminal human confirm remains available but is OPTIONAL and
NON-BLOCKING.

Purpose: The PTY e2e is the autonomous answer to the recurring D-16/D-13 blindspot —
it feeds RAW keystrokes through the real input decoder (which Update(msg)/teatest
bypass), so it catches the exact 'couldn't type in the wizard' regression that passed
every model test and failed only on a real terminal. The UI critique catches
design-contract drift (typography, color, glyph, keymap, copy, spacing) that automated
tests cannot judge; the code review enforces the phase's security invariants
(safe-write backup/atomic, gosec G204 arg-slice, no shell, no auto-delete of user
files, wiring-guard tests present). The optional human pass covers only the thin
residual slice a PTY emulator cannot reproduce (a specific terminal over SSH).
Output: 05.7-UI-CRITIQUE.md + 05.7-CODE-REVIEW.md with resolved findings.
</objective>
