# Plan 05.7-14 — Leg-1 correction: write the SSH alias block BEFORE the resolved test (UAT G-5, BLOCKER)

<objective>
Fix the inverted test/write order in the staged create wizard so the resolved/alias
SSH test can actually resolve. Per CLAUDE.md the contract is **test → confirm + backup
→ re-test the resolved configuration**. Today Phase 2 tests `ssh -T git@<alias>` BEFORE
the `Host <alias>` block is written, so any alias fails with "could not resolve hostname"
and the wizard can never complete. Also make the alias honest (blank → provider host, not
an invented `<name>.<provider>` suffix) and show the test command on every phase.
</objective>

<context>
Current flow (tui/wizard.go): initProve → Phase 1 (tester.PreWrite, explicit `-i <key>`
against the real hostname — needs NO block) → on PASS → Phase 2 (tester.Resolved:
`ssh -T git@<alias>` + `ssh -G <alias>` — REQUIRES the block) → on PASS → PersistSSH.
The block is written AFTER Phase 2, so Phase 2 always fails for an alias. The wizard
comment even claims "test that the WRITTEN alias resolves" — but nothing is written first.

internal/identity.PersistSSH already persists the key to ~/.ssh and writes the Host block
with a timestamped backup + idempotent managed block. internal/tester.PreWrite uses an
explicit key + real hostname (block-independent); tester.Resolved uses the alias.
</context>

<files>tui/wizard.go, tui/wizard_test.go</files>
<action>

against the alias (now resolvable via the written block) → show the Phase 2 outcome.

uploaded yet): it still writes the Host block (PersistSSH) so the alias exists for later.
