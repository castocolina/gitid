<objective>
LEG 1 — close the dangling-alias problem. When the create wizard does NOT write the
`~/.ssh/config` Host block (the brand-new key fails the SSH test because it is not on
the provider yet, or the user cancels), the wizard currently leaves the user with no
usable alias and no explanation. This plan makes the no-write state legible and makes
the path to a real, resolvable alias discoverable: upload the key first (AUTOUP-01
gh/glab assist) then retry, OR `[s]` skip-&-write-offline.

Depends on Plan 09 (same file tui/wizard.go — must run after the preview/legibility
landing). Closes (the remaining half of): G-2.

Purpose: User reported "Sigo sin ver el bloque en ~/.ssh/config que agrega el alias;
sin eso no funciona." The expected end state (recipes/) requires the Host alias block
to exist so `git push`/`clone` resolves. The wizard must either write it or tell the
user exactly how to get it written.
Output: A clear "nothing was written + why + two ways forward" feedback surface and a
discoverable, working skip-&-write-offline path.
</objective>
