# Phase 05.7 Plan 09: Alt-SSH Mapping + PersistSSH/PersistGitconfig Split Summary

One-liner: Centralized recipe alt-SSH mapping (DefaultHostname/DefaultPort) + PersistAll split into independently callable PersistSSH (LEG 1) and PersistGitconfig (LEG 2) composable steps.

## What Was Built

### Task 1: DefaultHostname / DefaultPort (D-10 parity)

Added two UI-free helpers to `internal/identity/identity.go` alongside the existing `DefaultAlias` and `DefaultMatch`:

- `DefaultHostname(provider string) string` — maps the three known providers to their recipe alt-SSH endpoints:
  - github -> `ssh.github.com`
  - gitlab -> `altssh.gitlab.com`
  - bitbucket -> `altssh.bitbucket.org` (was missing in cmd; falling back to `bitbucket.com`)
  - unknown without dot -> `<provider>.com` (cmd parity)
  - unknown with dot -> provider verbatim (cmd parity)
  - Lookup is case-insensitive; handles `"github.com"` by stripping the suffix token

- `DefaultPort() int` — returns 443 (single constant for cmd and TUI)

`cmd/gitid/add.go`'s `defaultHostname()` is now a thin delegate (`return identity.DefaultHostname(provider)`). Both `promptPort` call sites use `identity.DefaultPort()` instead of the literal `443`. No inline switch remains in `cmd/`.

Tests (`TestDefaultHostname`, `TestDefaultPort`) pin all three recipe endpoints, case-insensitive matching, dot-containing provider detection, and both fallback shapes.

### Task 2: PersistSSH + PersistGitconfig Split (T-05.7-09)

Decomposed `PersistAll` into two composable, independently callable steps:

**`PersistSSH(in, staged, deps)` — LEG 1:**
- Calls `deps.PersistKey(staged)` first when `staged.PrivPEM != nil` (before any Host block write — T-05.7-09-04 invariant preserved)
- Renders host block via `sshconfig.RenderHostBlock` (no hand-built strings)
- Calls `deps.WriteSSH`; returns `SSHBackup`
- Does NOT fire `WriteGitconfig`, `WriteFragment`, `WriteAllowedSigners`, or `Resolved`

**`PersistGitconfig(in, staged, deps)` — LEG 2:**
- Calls `deps.WriteGitconfig`, `deps.WriteFragment`, `deps.WriteAllowedSigners`
- Returns `GitconfigBackup`, `AllowedSignersBackup`, gitconfig/signers previews
- Does NOT touch the key or the Host block, does NOT run `Resolved`

**`PersistAll` (thin composition):**
- `res1 := PersistSSH(...)` then `res2 := PersistGitconfig(...)` then `deps.Resolved(...)`
- Merges via `mergeCreateResults` private helper; returned `CreateResult` is field-for-field equal to the prior monolith
- `cmd/gitid/add.go` `runCreateNew` is **unchanged** — it still calls `PersistAll`

Four tests assert the anti-blindspot requirements with call-log fakes:
- `TestPersistSSH_OnlyLEG1Writers` — PersistKey + WriteSSH fire; LEG-2 writers + Resolved do not
- `TestPersistSSH_SkipsPersistKeyWhenPrivPEMNil` — PersistKey skipped for existing-key path
- `TestPersistGitconfig_OnlyLEG2Writers` — WriteGitconfig + WriteFragment + WriteAllowedSigners fire; LEG-1 writers + Resolved do not
- `TestPersistAll_CompositionEqualsLEG1PlusLEG2` — writer counts + field-for-field equality against the reference composition
