<objective>
Build the UI-free `internal/adopter` package (ADOPT-01 core) test-first: detect
plain-style `~/.gitconfig_<name>` fragments via `filepath.Glob`, and adopt them by
MIGRATE (copy into `~/.gitconfig.d/<name>` + repoint the managed `includeIf`, D-04)
or REFERENCE-IN-PLACE (point `includeIf` at the existing flat file). Migrate is
copy-then-repoint and NEVER destructive (D-05) — the original is preserved and
removal is a separate explicit step the caller performs.

CRITICAL premise correction (REVIEWS.md #1): RESEARCH.md and CONTEXT D-06 state
adoption "reuses doctor's orphan detection." This is FALSE.
`internal/doctor/checks/orphans.go` detects only three classes — orphaned SSH
managed blocks, orphaned gitconfig managed blocks, and unreferenced KEY files
(via `AllSSHHostIdentityFiles`). `CheckOrphans` NEVER globs `~/.gitconfig_*` and
returns ZERO plain-style fragment candidates. Adoption candidate detection
therefore does NOT exist and MUST be built from scratch here as
`adopter.ListCandidates(homeDir, managedNames)` using `filepath.Glob`. Do not call
`doctor.CheckOrphans`; do not propagate the "reuse" premise into any task text.

Purpose: This is the orchestration layer that composes existing primitives
(`filewriter`, `gitconfig.WriteIncludeIf`, `gitconfig.ReadFragment`) PLUS the new
from-scratch fragment-glob detector. Keeping it UI-free makes it fully
unit-testable and reusable by both the TUI adopt modal (Plan 07) and the
`gitid adopt` CLI command (Plan 06).
Output: GREEN `internal/adopter` package replacing the Plan 01 RED stub.
</objective>
