## Finding 2 ‚Äî gosec G204 arg-slice: no `sh -c` anywhere [PASS]

**Files inspected:** `internal/repoclone/repoclone.go`, `internal/uploader/uploader.go`, `tui/sidebar.go`

All subprocess invocations use `exec.Command(binary, arg, arg, ...)` ‚Äî never `exec.Command("sh", "-c", ‚Ä¶)` or `exec.Command("bash", "-c", ‚Ä¶)`.

| Call site | Binary | Form |
|-----------|--------|------|
| `repoclone.liveClone` | `"git"` | `exec.Command("git", "clone", "--", cloneURL, destPath)` |
| `repoclone.livePull` | `"git"` | `exec.Command("git", "-C", destPath, "pull")` |
| `uploader.buildArgs ‚Üí RunCmd` | `"gh"` / `"glab"` | `exec.Command(toolPath, "ssh-key", "add", pubPath, ‚Ä¶)` |
| `sidebar.go openFinder` | `"open"` / `"xdg-open"` | arg-slice form |
| `sidebar.go openTerminal` | `"open"` / `xdg-open` | arg-slice form |

The `"--"` separator in `git clone -- <url> <dest>` (added in commit `65d3548` as the flag-smuggling defense-in-depth fix) is present and annotated. This closes the argv injection vector where a crafted `cloneURL` starting with `-` could be interpreted as a git option.

All `G204` annotations include the rationale (arg-slice, no shell, specific guard applied).

**Verdict:** PASS ‚Äî G204-clean throughout.

```go
func ListCandidates(homeDir string, managedNames []string) ([]string, error) {
    pattern := filepath.Join(homeDir, ".gitconfig_*")
    matches, err := filepath.Glob(pattern)
```

The pattern is constructed from `filepath.Join` (not string concatenation), preventing path separator injection. The pattern `.gitconfig_*` is a fixed prefix glob ‚Äî it does NOT traverse directories (no `**`) and does NOT follow symlinks (confirmed: the lstat call in `Adopt` uses `os.Lstat` which does not follow symlinks, per G304 annotation with justification).

The `G304` annotation on the `os.Lstat` call explains: `sourcePath is a gitid-derived candidate path returned by filepath.Glob` ‚Äî this is the correct level of specificity for a gosec suppression.
