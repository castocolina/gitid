<objective>
Establish the HTML/MUI-v7 mockup foundation (DLV-01): a Vite + React 19 + TypeScript + MUI v7.3.11 SPA under `.planning/design/mockup-src/`, themed into a **terminal skin** so the HTML↔TUI parity gate is semantic (not pixel noise), plus the shared app shell (header / body / status line / keybar) every one of the seven surfaces renders inside, plus a typed recipe-accurate copy fixtures module, plus glob-based route auto-discovery (with build-time route-uniqueness/shape validation) so surface plans ADD route files without editing shared code. Purpose: DLV-01 requires an HTML mockup BEFORE any Go/TUI code for a surface; the mockup must lock content + flow (fields, labels, copy, order, states). A terminal-skinned shell + recipe fixtures make every later mockup fast, consistent, and recipe-faithful. Auto-discovery keeps the six-surface fan-out (Wave 4) conflict-free.
Output: the pnpm workspace, the terminal-skin theme, the shell components (incl. the breadcrumb screen-ID marker), the recipe fixtures module, glob route discovery + validation, and the three design-doc templates (FIELDS/CRITIQUE/PARITY).
