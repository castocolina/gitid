zones), the TUI-demo changes (first-class stepper, focused/blurred field contours, stable hint
zone, bounded previews, dimmed disabled nav), and all gates re-run green.

**Two implementation traps the reviews quantified — obey them or the plan regresses:**
1. **Row budget (HIGH).** A rounded border on all 6 SSH fields ≈ +12 rows and CANNOT fit the
   100×30 frame the demo was compacted into across four documented viewport rounds. Only the
   FOCUSED field gets the full rounded box (+2 rows); blurred fields get a single-row dim
   contour. Net cost ~+2 rows, not +12. The 100×30 PTY e2e must still pass.
2. **←/→ is a triply-claimed key (HIGH).** It already moves the field/button ring, changes the
   strategy select, and switches Global-SSH sub-tabs. Wizard step-nav is added under an EXPLICIT
   precedence rule written into 02-STYLE-SPEC.md and implemented identically in both media —
   never stealing cursor keys from a focused text input.

machine files (the "63 rows") were REMOVED with the static reference set in commit 7453561.
Parity is now enforced by (a) the Go copy-pinning unit suite in `internal/dummytui`, (b) the
per-surface `FIELDS.md`/`CRITIQUE.md` docs, and (c) a fresh `agent-ui-ux-designer` critique of
the two LIVE demos. This plan EXTENDS parity with the new emphasis-role dimensions recorded in
02-STYLE-SPEC.md and backed by new Go tests — do NOT hunt for a 63-row JSON file; it is gone.
