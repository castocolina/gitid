<objective>
Build the navigable Go TUI **dummy** skeleton (DLV-05 front half): `internal/dummytui` + a physically separate `cmd/gitid-dummy` binary that renders the SAME four-region shell as the MUI mockup and routes the five primary views on number keys `1`–`5` plus modal entry points — with ZERO backend logic, enforced as an **allowlist** import-graph property (`go list -deps`), not a denylist and not a promise. Lands the contracts surfaces plug into: a registry with `Register` (rejects duplicate activation keys) and `RegisterOrReplace` (claims/replaces a key so fan-out surfaces replace placeholders without a conflict, never editing model.go/data.go), a deterministic full-shell `RenderScreen` (incl. the `<surface>/<screen>` breadcrumb screen-ID) for capture, and a backend-free modal-overlay helper (lipgloss v2.0.3 has no `PlaceOverlay`). Ships only placeholder screens for the five views.

Purpose: DLV-05 fixes the per-surface order HTML→dummy→approval→backend and mandates no backend logic. The separate binary + import ALLOWLIST make "no backend" an enforced CI signal that catches NEW/renamed backend packages (a denylist cannot); the replacement semantics keep the Wave-4 fan-out conflict-free.
Output: `internal/dummytui/{doc,registry,model,shell,overlay,data}.go` + tests, `cmd/gitid-dummy/main.go`, the `nobackend` allowlist test.

Review folded in (Codex 02-REVIEWS.md): HIGH-1 flip no-backend from denylist → ALLOWLIST (fail on any first-party pkg != dummytui/gitid-dummy); HIGH-2 add `RegisterOrReplace` so fan-out 06–10 don't collide on keys 1–5; HIGH-3(b) RenderScreen renders the full shell incl. the `<surface>/<screen>` breadcrumb so tests assert the SCREEN, not just a signature.

TDD: the nav state machine (screen enum + key routing; number-key → view; modal open/close; register vs register-or-replace) is real logic with defined I/O — Tasks 1–2 are RED→GREEN→REFACTOR. Shell rendering + overlay port are execute-style (static View()).
