<objective>
Build the navigable Go TUI **dummy** skeleton (DLV-05 front half): a new `internal/dummytui` package + a physically separate `cmd/gitid-dummy` binary that renders the SAME four-region shell as the MUI mockup and routes the five primary views on number keys `1`–`5` plus modal entry points — with **ZERO backend logic**, enforced as a grep-checkable import-graph property (`go list -deps`), not a promise. It also lands the reusable contracts the surfaces plug into: a surface **registry** (surfaces register their screens + key transitions + a deterministic `RenderScreen` in their own file, never editing `model.go`) and a backend-free modal-overlay compositing helper (lipgloss v2.0.3 has no `PlaceOverlay`). This plan ships only placeholder screens for the five views; the pilot and fan-out surfaces fill real content.

Purpose: DLV-05 fixes the per-surface order HTML→dummy→approval→backend, and mandates the dummy carry no backend logic. A separate binary + import-graph proof makes "no backend" an enforced CI signal, and the registry keeps the six-surface fan-out (Wave 4) conflict-free.
Output: `internal/dummytui/{doc,registry,model,shell,overlay,data}.go` + tests, `cmd/gitid-dummy/main.go`, and the `nobackend` import-graph test.

TDD: the navigation state machine (screen enum + key routing → next screen; number-key → view; modal open/close) is real logic with defined I/O — `expect(route(state, key)) == nextState`. Tasks 1–2 are RED→GREEN→REFACTOR. The shell rendering + overlay port are `type: execute` (static View() output, no meaningful RED).

DLV-02: this is a UI task. The dummy shell + screens must mirror the `/mui`-built HTML mockup's regions/copy (the `/mui` mockup is the parity source), and `agent-ui-ux-designer` validates the terminal-skin cross-media shell parity (UX-DIRECTION §2). Name both explicitly.
</objective>
