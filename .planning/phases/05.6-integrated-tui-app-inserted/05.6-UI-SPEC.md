# Phase 05.6 — Terminal TUI Interaction Contract

> This phase has no graphical or web UI. This document is a **terminal TUI
> interaction contract** for a Bubble Tea v2 / Lipgloss v2 / Bubbles v2
> application running over SSH. All six UI-SPEC dimensions are mapped to
> terminal idioms.
>
> - **Spacing/layout** → lipgloss padding/margin values, pane widths, column
>   grid, responsive breakpoint
> - **Typography** → emphasis levels (bold/faint/underline/reverse), hierarchy
>   conventions, truncation rules — not web fonts
> - **Color** → ANSI-indexed palette expressed via Lipgloss v2 adaptive color
>   with manual downsampling; must degrade on 256-color, 16-color, and
>   monochrome SSH terminals
> - **Copywriting** → screen titles, CTA labels, footer key hints, empty/
>   loading/error/confirm/destructive copy
> - **Design system** → reusable Lipgloss v2 style tokens + Bubbles v2 widget
>   inventory, shared keymap
> - **Accessibility** → NO_COLOR / non-TTY plain mode is the accessibility
>   contract; glyph+color pairing (never color alone)
>
> **What this spec replaces:**
> Phase 05 UI-SPEC (`05-UI-SPEC.md`) described a screen-stack TUI
> (Dashboard → Identity List → Identity Detail → forms). Phase 5.6 replaces
> the screen-stack architecture with a **persistent two-pane layout** (sidebar
> always visible) plus modal overlays. Styles, color tokens, glyph contract,
> and keymap primitives are inherited and extended, not replaced.
>
> **Sources for pre-populated decisions:**
> - `05.6-CONTEXT.md` D-01..D-16 (locked layout, editing, health, and build
>   decisions)
> - `05-UI-SPEC.md` (prior Phase 5 contract — color tokens, glyph contract,
>   style tokens, keymap; adopted with extensions)
> - `05.5-EXPERIENCE-EVAL.md` §Handoff — 7 UX learnings, especially #2
>   (warnings must be visually distinct from errors)
> - `VERIFICATION-PLAYBOOK.md` §Target TUI App (lines 779-811)
> - `CLAUDE.md` Technology Stack table (locked Charm v2 import paths)

## Persistent Layout Architecture

> Source: 05.6-CONTEXT.md D-01, D-02, D-03, D-04 (all LOCKED)

### Base Layout (above responsive breakpoint)

```
┌─────────────────────────────────────────────────────────────────────────┐
│ HEADER: gitid  ·  [Identities]  Health  Global Options      ● 2 issues │
├──────────────────────┬──────────────────────────────────────────────────┤
│ SIDEBAR              │ MAIN PANE (master-detail / view content)         │
│ (18 cols fixed)      │ (remaining width)                                │
│                      │                                                  │
│ Identities           │                                                  │
│ › personal  github 2 │                                                  │
│   work      gitlab 1 │                                                  │
│                      │                                                  │
│ Unmanaged            │                                                  │
│   (host-block-A) ~   │                                                  │
│   (key-orphan-B) ~   │                                                  │
│                      │                                                  │
├──────────────────────┴──────────────────────────────────────────────────┤
│ FOOTER: q quit  tab focus  ↑↓ move  e edit  c copy  a add  ? help      │
└─────────────────────────────────────────────────────────────────────────┘
```

**Header row:** 1 row. Contents: `gitid` (bold) + spacer + view tabs (active tab
bold+underline, inactive tabs faint) + right-aligned global health badge.

**Sidebar:** 18 columns fixed width. Permanently visible (D-01). Two sections:
managed identities (interactive) and read-only unmanaged entries (D-12).

**Main pane:** fills `terminalWidth - 18 - 3` columns (subtract sidebar width +
border cells). Displays the active view's content (Identities detail, Health,
Global Options).

**Footer:** 1 row. Bold key tokens, comma-separated. Always rendered (D-04, closes
G-02/G-04). Rendered via `bubbles/v2 help.Model` in compact mode.

**Between sidebar and main pane:** a `lipgloss.NormalBorder()` vertical separator.

### Responsive Collapse (D-03)

**Breakpoint:** terminal width < 80 columns → collapse to single-pane mode.

In collapsed mode:
- Sidebar is hidden. Only the active pane fills the screen.
- A persistent sidebar-toggle key `\` (backslash) reappears the sidebar as a
  temporary overlay (full height, same 18-col width, over the content).
- If the terminal is also < 24 rows: render `Terminal too small — resize to at
  least 80×24` in plain text (no lipgloss) and halt TUI rendering.
- Header still shows the view tabs and health badge.
- Footer still shows key hints including `\` with description `toggle sidebar`.

**Width detection:** `tea.WindowSizeMsg` on startup and on every subsequent resize
event; re-evaluate breakpoint and re-layout on each message.

### Modal Overlay (D-02)

Create/add wizard, delete confirm, rotate confirm, fix confirm, and help overlay
render as centered modal boxes over the dimmed persistent layout:

```
┌─────────────────────────────────────────────────────────────────────────┐
│ HEADER (dimmed)                                                          │
├──────────────────────┬──────────────────────────────────────────────────┤
│ SIDEBAR (dimmed)     │ MAIN PANE (dimmed)     ╔══════════════╗          │
│                      │                        ║  MODAL TITLE ║          │
│                      │                        ║              ║          │
│                      │                        ║  [fields]    ║          │
│                      │                        ║              ║          │
│                      │                        ╚══════════════╝          │
├──────────────────────┴──────────────────────────────────────────────────┤
│ FOOTER: esc cancel  enter confirm                                        │
└─────────────────────────────────────────────────────────────────────────┘
```

**Dimming:** apply `lipgloss.NewStyle().Faint(true)` to the behind-modal content
render. The persistent layout stays visible but visually recedes.

**Modal box styling:** `lipgloss.NewStyle().Border(lipgloss.RoundedBorder()).
BorderForeground(lipgloss.Color("4"))`. Max width: `min(terminalWidth-8, 72)` cols.
Center with `lipgloss.Place()`.

## Spacing Scale

> Terminal equivalent: lipgloss padding/margin values and blank-line rhythm.
> All values are character cells (columns/rows), not pixels.

| Token | Lipgloss Value | Terminal Usage |
|-------|---------------|----------------|
| xs | 0 | Between tightly-packed items (no gap); glyph-to-text on same line |
| sm | 1 | Inner padding of sidebar rows; horizontal gap label→value; icon→text |
| md | 2 | Panel internal padding left/right; vertical gap between sections |
| lg | 3 | Outer modal margin; space above screen title in header |
| xl | 1 blank row | Between major sections in the main pane (e.g. fields block vs health block) |
| code | 4 | Indent for rendered code/config blocks (e.g. the live `includeIf` preview); matches the Phase 4 CLI 4-space code-block convention |

**Concrete per-element rules:**

| Element | Padding/Margin |
|---------|---------------|
| Sidebar row | `PaddingLeft(1)` for indent; `PaddingLeft(3)` for focused-indicator row |
| Header | `Padding(0, 2)` (no vertical; 2-col left/right) |
| Footer | `PaddingTop(1)` + top divider border |
| Main pane | `Padding(1, 2)` (1 row top/bottom, 2 cols left/right) |
| Modal box | `Padding(1, 2)` internally |
| Form field row | `Padding(0, 1)` + `MarginBottom(1)` between fields |
| Doctor family panel | `Padding(1, 2)` |
| Inline edit field (active) | No extra padding; the border provides visual separation |
| includeIf preview block | `PaddingLeft(4)` (matches Phase 4 CLI 4-space convention) |

## Typography

> Terminal equivalent: text emphasis via ANSI attributes (bold/faint/underline/
> reverse video). One typeface: the terminal's monospace font.

| Role | Style Token | ANSI Attribute | Usage |
|------|-------------|----------------|-------|
| App name / view tab (active) | `StyleTitle` | Bold | `gitid` in header; active view tab label |
| View tab (inactive) | `StyleTabInactive` | Faint | Inactive view tab labels in header |
| Section header | `StyleHeader` | Bold | Family name in Health view; sidebar section headers (`Identities`, `Unmanaged`) |
| Selected item | `StyleSelected` | Bold + Reverse video | Focused row in sidebar or list |
| Body / primary | `StyleBody` | Default foreground | Identity name in detail, field values, explanation text |
| Secondary / de-emphasized | `StyleFaint` | Faint | Key paths, secondary metadata (port, alias), suggested-fix text, unmanaged entries |
| Field label | `StyleLabel` | Bold, fixed width 16 | Form field labels; metadata key in detail view (`Name:`, `Email:`, etc.) |
| Passing indicator | `StylePass` | ColorPass foreground | `✓` glyph + passing description |
| Finding / severity text | `StyleFinding` | Severity color foreground (applied per call) | `✗` / `!` / `~` glyph + finding title |
| Active input field | `StyleInputActive` | ColorAccent border (rounded) | Text input when focused |
| Inactive input field | `StyleInputInactive` | ColorMuted border (normal) | Text input when blurred |
| Footer key | `StyleHelpKey` | Bold + Faint | Key tokens in footer: `q`, `tab`, `↑↓` |
| Footer description | `StyleHelpDesc` | Faint | Key descriptions: `quit`, `focus`, `move` |
| Read-only field | `StyleReadOnly` | Faint + Italic (where terminal supports) | Immutable fields (e.g. `Name (immutable):` in update form) |
| Modal title | `StyleModalTitle` | Bold | First line inside a modal box |
| Badge (health, sidebar) | severity glyph + `SeverityStyle(s)` | Severity color | Per-identity health badge in sidebar row; global badge in header |

**Truncation rules:**

| Context | Rule |
|---------|------|
| Sidebar identity row | Truncate name at `sidebarWidth - 10` cols with `…`; provider + count fill remaining cols |
| Main pane field value | Truncate at `mainPaneWidth - 20` cols with `…`; full value shown in viewport scroll |
| Header title | Never truncate; `gitid` + tabs always fit at ≥ 80 cols |
| Health view finding title | Truncate at `mainPaneWidth - 6` cols with `…`; `f` key or Enter expands |
| Footer key hint line | Truncate entire footer to `terminalWidth - 2` cols; prefer dropping rightmost hints |
| includeIf preview | Never truncate; scroll horizontally in viewport if needed |
| Modal title | Fixed at `modalWidth - 4` max; center-aligned |

**Label column width:** fixed at 16 characters, right-padded with spaces; values
fill remaining pane width.

## Color

> Severity-driven palette. ANSI-indexed colors only (no hex, no TrueColor
> requirement) so the palette degrades correctly on 256-color, 16-color, and
> monochrome SSH terminals via Lipgloss v2's adaptive color behavior.
>
> **Source:** 05-UI-SPEC.md color tokens (inherited verbatim) + new tokens for
> the two-pane header/tab/badge additions.

### Mandatory Downgrade Rules (in precedence order)

1. `NO_COLOR` env var set (any non-empty value) → disable all lipgloss color;
   plain text only.
2. Non-TTY stdout (piped / CI) → disable all lipgloss color; plain text only.
3. `$TERM == "dumb"` or `$TERM` unset → ASCII glyph fallbacks; no ANSI codes.
4. TTY with 256-color support → full ANSI-indexed palette below.
5. TTY with 16-color only → lipgloss auto-downsampling maps to nearest ANSI-16.

**Implementation:** `lipgloss.NewStyle()` (NOT `renderer.NewStyle()`) used
throughout — lipgloss v2's internal renderer detects the terminal profile
automatically from `os.Stdout`. For the `NO_COLOR` / non-TTY check, the existing
`colorEnabled` / `os.ModeCharDevice` pattern from Phase 4 applies.

### Color Token Table

| Token name | ANSI index | Lipgloss call | Usage |
|------------|-----------|---------------|-------|
| `ColorCritical` | 1 (red) | `lipgloss.Color("1")` | Critical finding glyph + title; critical badge |
| `ColorError` | 1 (red) | `lipgloss.Color("1")` | Error finding glyph + title |
| `ColorWarning` | 3 (yellow) | `lipgloss.Color("3")` | Warning finding glyph `!` + title; warning badge — **DISTINCT from error** (Eval #2) |
| `ColorInfo` | 6 (cyan) | `lipgloss.Color("6")` | Info finding glyph `~` + title; info badge |
| `ColorPass` | 2 (green) | `lipgloss.Color("2")` | Passing check `✓` glyph + text; pass badge |
| `ColorFaint` | default | `.Faint(true)` | Secondary metadata; suggested-fix text; unmanaged entries |
| `ColorBold` | default | `.Bold(true)` | Section headers; active view tab |
| `ColorAccent` | 4 (blue) | `lipgloss.Color("4")` | Active input border; focused panel border; `›` sidebar indicator; modal border; spinner |
| `ColorMuted` | 8 (bright black) | `lipgloss.Color("8")` | Footer help text; inactive input border; inactive tab labels; placeholder text; sidebar separator |
| `ColorDimOverlay` | default | `.Faint(true)` | Behind-modal content dimming (applied to entire persistent layout render) |

### 60/30/10 Terminal Split

- **60% dominant:** default terminal foreground — body text, identity names,
  field values, explanation text, modal content.
- **30% secondary:** `ColorFaint` / `ColorMuted` — fix text, key paths, footer
  bar, unmanaged section entries, inactive view tabs, form placeholders.
- **10% accent:** severity colors + `ColorAccent` — reserved exclusively for
  the elements listed below.

### Accent Reserved For (explicit list — do not use elsewhere)

1. `✓` glyph + passing text in the Health view and sidebar badge.
2. `✗` glyph + title for critical/error findings in the Health view.
3. `!` glyph + title for warning findings in the Health view (yellow, distinct
   from `✗` — this fixes Eval #2 / F4.3).
4. `~` glyph + title for info findings in the Health view.
5. Active/focused text input border in forms and inline edit.
6. Focused pane border (sidebar OR main) when using `Tab` cycle.
7. `›` navigation indicator on the selected sidebar row.
8. Modal box border (create wizard, confirm dialogs, help overlay).
9. Spinner animation in Health view and wizard progress.
10. Active view tab in the header (bold, not color — `ColorBold`).
11. Global health badge in header (severity color of worst open finding).

**Colorblind-safe rule (mandatory):** color is never the sole carrier of meaning.
Every severity level is also conveyed by a glyph (`✓` `✗` `!` `~`) AND a text
label (`[critical]` / `[warning]` / `[info]`) inline in finding titles.

## Glyph Contract

> Inherited from Phase 4 and Phase 05 UI-SPEC. Extended with sidebar and badge
> glyphs specific to the two-pane layout.

| Semantic | Glyph (UTF-8) | ASCII fallback | Color | When used |
|----------|--------------|----------------|-------|-----------|
| Passing check | `✓` | `OK` | `ColorPass` (green) | Health view — check passed; sidebar badge; detail health summary |
| Failing — critical/error | `✗` | `FAIL` | `ColorError` (red) | Health view — critical or error finding; badge in sidebar |
| Advisory — warning | `!` | `!` | `ColorWarning` (yellow) | Health view — warning finding; badge (distinct from `✗`) |
| Advisory — info | `~` | `i` | `ColorInfo` (cyan) | Health view — info finding; sidebar unmanaged entry prefix |
| Fixable marker | `[fix]` | `[fix]` | default (bold) | Inline after a fixable finding line |
| Selected row | `›` | `>` | `ColorAccent` (blue) | Focused/selected row in sidebar and any list |
| Bullet (multi-value) | `•` | `*` | default | Multi-value metadata (aliases list, sites list) |
| Loading | spinner chars | `...` | `ColorAccent` | Family loading in Health view; wizard test phases |
| Incomplete identity | `?` | `?` | `ColorWarning` | Sidebar badge: identity reconstructed with missing artifacts |
| Unmanaged entry prefix | `○` | `o` | `ColorFaint` | Sidebar unmanaged section rows (hand-written Host block or orphan key) |
| Identity site count | `(N)` | `(N)` | `ColorFaint` | Sidebar row trailing: number of aliases/sites |
| Inline edit active | `[` `]` | `[` `]` | `ColorAccent` border | Visual hint that a field is in inline-edit mode |

**Glyph selection rule:** use UTF-8 glyphs unless `$TERM == "dumb"` or `$TERM`
is unset. Non-TTY piped output does NOT degrade glyphs (consistent with Phase 4).
Color degrades independently of glyphs.

### View 1: Identities (default on launch)

**Header tab label:** `Identities` (active: bold; inactive: faint)

**Sidebar content:**
- Section label `Identities` (`StyleHeader`, bold)
- One row per managed identity: `› name  provider  (N)` where `N` is alias/site
  count. Row format: `StyleSelected` on focused row; `StyleBody` on others.

- Section separator: 1 blank row
- Section label `Unmanaged` (`StyleHeader`, faint — read-only signal)
- One row per unmanaged entry: `○ <short-name> ~` (`StyleFaint`)
  Where `<short-name>` is the first 12 chars of the Host alias or key filename.
- Empty unmanaged section: omit the section entirely (no noise when clean).

**Main pane content — no identity selected (empty state):**
```
  No identities configured.
  Press 'a' or Ctrl+P to create your first identity.
```
`StyleBody` for first line; `StyleFaint` for second line. Vertically centered in
the pane.

**Main pane content — identity selected (detail view):**

```
  personal                                         ✓ all checks pass

  Name:             personal
  Git Name:         Ramon Colina
  Git Email:        user@example.com
  Provider:         github.com
  Port:             22

  Aliases / Sites:
  • personal.github.com     gitdir:~/git/personal/
  • work.github.com         hasconfig:remote.*.url:git@github.com:work/**

  Signing:          enabled
  Key Path:         ~/.ssh/gitid_personal

  Health:
    ✓ all checks passed
```

- Identity name: `StyleTitle` (bold), top left.
- Health badge top right: severity glyph + `all checks pass` or worst-finding
  summary, right-aligned to pane width.
- Label column: `StyleLabel` (bold, 16 chars fixed width).
- Value column: `StyleBody`.
- Key path: `StyleFaint`.
- `Aliases / Sites:` header: `StyleHeader` (bold).
- Each alias row: `•` bullet + alias + 2-space gap + match condition
  (`StyleFaint` for the condition).
- `Health:` block: findings listed using glyph + severity color per finding;
  `✓ all checks passed` in `StylePass` when clean.

**Inline editing (D-05):**
Press `e` to enter inline edit mode. The focused field receives an
`StyleInputActive` border (blue, rounded). `Tab`/`Shift+Tab` cycle fields.
`Enter` commits the field. `Esc` cancels all edits. A visual indicator
`[editing]` appears in the header right area when in edit mode.

For structural fields (alias, hostname, port, match strategy): on `Enter`
a modal overlay for the prove-before-write test loop opens before writing.
For non-structural fields (git email, signing toggle): a simple confirm
modal opens (`Write changes? [Enter / Esc]`).

**Match strategy inline control (D-06):**
When the match strategy field is focused in inline edit mode, it expands to
show a live `includeIf` preview below the field:

```
  Match Strategy:   [gitdir:~/git/personal/          ]  (tab to change type)

  Preview:
    [includeIf "gitdir:~/git/personal/"]
        path = ~/.gitconfig.d/personal
```

The preview updates in real time as the user types. `PaddingLeft(4)` for the
preview block.

**Footer for this view (selected identity, not editing):**
```
q quit  tab focus  ↑↓/jk move  e edit  c copy  a add  d delete  ? help
```

**Footer for this view (no identity selected):**

## Accessibility Contract

1. **No information conveyed by color alone.** Every severity level uses both a
   color AND a distinct glyph (`✓` `✗` `!` `~`). This is a hard rule, including
   the warning vs. error distinction fixed in this phase.
2. **Piped output is plain text.** `gitid doctor 2>&1 | grep FAIL` works.
3. **NO_COLOR is respected.** Any non-empty `NO_COLOR` value strips all ANSI
   codes and lipgloss color.
4. **All prompts show the safe default visually.** `[Enter / Esc]` — Esc is
   always the safe default (no write, no deletion). The confirm action requires
   an affirmative Enter or explicit `y`.
5. **All destructive actions state consequences before confirm.** The delete,
   rotate, skip-test, and fix modals all render the consequence statement before
   the confirm prompt line.
6. **Footer always visible.** The compact key-hint bar is always rendered at
   the bottom of every view and modal.
7. **Minimum terminal size check.** Terminal < 80×24: render
   `Terminal too small — resize to at least 80×24` in plain text and halt TUI
   rendering (no garbled layout).
8. **English only.** All output, labels, prompts, error messages (CLAUDE.md).
9. **Keyboard-first.** Every action reachable without a mouse (D-04).

```go
    StyleFaint        = lipgloss.NewStyle().Faint(true)
    StyleLabel        = lipgloss.NewStyle().Bold(true).Width(16)
    StyleReadOnly     = lipgloss.NewStyle().Faint(true)

    StylePass    = lipgloss.NewStyle().Foreground(lipgloss.Color("2"))
    StyleFinding = lipgloss.NewStyle()  // color applied per-severity at call site

    StyleInputActive = lipgloss.NewStyle().

        BorderForeground(lipgloss.Color("4"))
    StyleInputInactive = lipgloss.NewStyle().
        Border(lipgloss.NormalBorder()).
        BorderForeground(lipgloss.Color("8"))

    // Footer / help
    StyleHelpKey  = lipgloss.NewStyle().Bold(true).Faint(true)
    StyleHelpDesc = lipgloss.NewStyle().Faint(true)

    // Panels
    StylePanel = lipgloss.NewStyle().
        Padding(1, 2).
        Border(lipgloss.RoundedBorder()).
        BorderForeground(lipgloss.Color("8"))

    StylePanelFocused = lipgloss.NewStyle().
        Padding(1, 2).
        Border(lipgloss.RoundedBorder()).
        BorderForeground(lipgloss.Color("4"))

    // Modal overlay box
    StyleModal = lipgloss.NewStyle().
        Padding(1, 2).
        Border(lipgloss.RoundedBorder()).
        BorderForeground(lipgloss.Color("4"))

    // Behind-modal dimming: apply to entire persistent layout render string
    StyleDimmed = lipgloss.NewStyle().Faint(true)

    // Sidebar
    StyleSidebarSection  = lipgloss.NewStyle().Bold(true)
    StyleSidebarItem     = lipgloss.NewStyle()
    StyleSidebarUnmanaged = lipgloss.NewStyle().Faint(true)
    StyleSidebarBadge    = lipgloss.NewStyle()  // foreground applied per severity
)

// SeverityStyle returns a lipgloss foreground style for the given severity.
// Critical and error both use red (ANSI 1). Warning uses yellow (ANSI 3)
// — distinct from error, fixing Eval #2 (F4.3). Info uses cyan (ANSI 6).
func SeverityStyle(s doctor.Severity) lipgloss.Style {
    switch s {
    case doctor.SeverityCritical, doctor.SeverityError:
        return lipgloss.NewStyle().Foreground(lipgloss.Color("1"))
    case doctor.SeverityWarning:
        return lipgloss.NewStyle().Foreground(lipgloss.Color("3"))
    default: // SeverityInfo
        return lipgloss.NewStyle().Foreground(lipgloss.Color("6"))
    }
}

// SeverityGlyph returns the glyph for the given severity.
// Warning uses "!" (not "✗") to distinguish it from errors.
// ASCII fallback used when term is dumb (caller checks $TERM).
func SeverityGlyph(s doctor.Severity, asciiMode bool) string {
    if asciiMode {
        switch s {
        case doctor.SeverityCritical, doctor.SeverityError:
            return "FAIL"
        case doctor.SeverityWarning:
            return "!"
        default:
            return "i"
        }
    }
    switch s {
    case doctor.SeverityCritical, doctor.SeverityError:
        return "✗"
    case doctor.SeverityWarning:
        return "!"
    default: // info
        return "~"
    }
}
```

### TUI Navigation / State Machine

```
App shell (persistent two-pane layout)
│
├── View 1: Identities (default)
│   ├── Sidebar: managed list + unmanaged list
│   └── Main: detail pane (selected identity fields, aliases, health)
│       ├── [e] Inline edit → field-by-field edits → commit/cancel
│       │   ├── Non-structural → simple confirm modal → write
│       │   └── Structural → prove-before-write modal → write
│       ├── [a] Create wizard modal (overlay)
│       │   ├── Step 1: form fields
│       │   ├── Step 2: key generation spinner
│       │   ├── Step 3: upload instructions + key display
│       │   └── Step 4: test loop → PASS → write / retry / skip / quit
│       ├── [d] Delete confirm modal (overlay) → write / cancel
│       ├── [R] Rotate confirm modal → Step 2-4 wizard → write / cancel
│       └── [c] Copy pubkey modal (overlay)
│
├── View 2: Health
│   ├── Sidebar: unchanged (last-known badges)
│   └── Main: 7 doctor families streaming async
│       └── [x] Fix confirm modal (overlay) → apply → re-run affected family
│
├── View 3: Global Options
│   ├── Sidebar: unchanged
│   └── Main: baseline fields (read-only + inline-edit)
│
├── [ctrl+p] Command palette modal (overlay, all views)
└── [?] Help overlay modal (all views)
```

**Modal stack rule:** at most 1 modal open at a time. Opening a second modal
(e.g. fix confirm while a delete confirm is open) is not permitted; the active
modal must be dismissed first. Implement as a single `activeModal` field in the
root model (not a stack).
