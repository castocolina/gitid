# Phase 05.6 Plan 01: Wave-0 Foundation Summary

Wave-0 foundation for the Phase 5.6 integrated TUI rewrite: overlay primitive verified, shared scaffolding extended, screen-stack architecture retired, and six RED test scaffold files laid down so per-slice plans start from a genuine failing baseline.

## What Was Built

### Task 1+2: Overlay Primitive + Extended Scaffolding (one logical commit)

**PlaceOverlay spike result (CONFIRMED):** `go doc charm.land/lipgloss/v2 PlaceOverlay` returns "no symbol PlaceOverlay in package charm.land/lipgloss/v2". Only `Place`, `PlaceHorizontal`, `PlaceVertical` exist. Recorded as `const placeOverlayAvailable = false` and in the overlay.go top-of-file doc comment.

**tui/overlay.go:**
- `placeOverlay(x, y int, fg, bg string) string` — line-by-line string compositing of a modal box over a dimmed background. Replaces background cells column-by-column using lipgloss.Width for ANSI-aware visible-width measurement; operates at rune level (never splits multi-byte glyphs or ANSI escapes). Oversized modals clamp silently (T-05.6-02).
- `modalOrigin(termW, termH, modalW, modalH int) (x, y int)` — centered origin with floor-at-zero clamping.
- `modalBox(width int, title, body string) string` — styled modal box, width clamped to `min(width-8, 72)` per UI-SPEC.

**tui/styles.go extended:**
- 10 new style tokens: `StyleTabActive`, `StyleTabInactive`, `StyleReadOnly`, `StyleModalTitle`, `StyleModal`, `StyleDimmed`, `StyleSidebarSection`, `StyleSidebarItem`, `StyleSidebarUnmanaged`, `StyleSidebarBadge`.
- `SeverityGlyph(s doctor.Severity, ascii bool) string`: warning → `"!"`, error/critical → `"✗"`, info → `"~"`. Warning is NEVER `"✗"` — this implements D-10/Eval#2 (distinct glyph per severity).
- `asciiMode() bool`: returns true when `$TERM == "dumb"` or `$TERM` unset.

**tui/keymap.go extended:**
- 10 new bindings: `Palette`/ctrl+p, `View1`/`View2`/`View3` (1/2/3), `SidebarToggle`/`\`, `Focus`/tab, `FocusRev`/shift+tab, `Fix`/x, `Retry`/r, `Skip`/s.
- Tab routing note (Pitfall 9) documented: `Focus` and `Next` share "tab"; root model routes by `activeModal != noModal` (Plan 02 wires this).

**tui/messages.go extended:**
- Retired `pushScreenMsg`, `popScreenMsg`, `pushCmd`, `popCmd` (screen-stack, D-15).
- 8 new message types: `clearModalMsg`, `refreshSidebarMsg`, `setToastMsg`, `clearToastMsg`, `deleteResultMsg`, `fixResultMsg`, `rotateResultMsg`, `baselineLoadedMsg`.
- 4 helper cmd constructors: `clearModalCmd()`, `refreshSidebarCmd()`, `setToastCmd()`, `clearToastAfter()`.

**Screen-stack retirement (D-15 clean rewrite):**
- 9 source files deleted: identitylist.go, identitydetail.go, createform.go, updateform.go, addaccountform.go, dashboard.go, prove.go, copy.go, old model.go.
- 6 test files deleted: dashboard_test.go, model_test.go, navigation_test.go, prove_test.go, form_test.go, reviewfix_test.go.
- Git history preserves all originals; Plans 02-06 read them via `git show HEAD:tui/<file>` when porting.
- Minimal `model.go` stub created: `tuiDeps` struct + `rootModel` stub + `newRootModel` so `tui.go` and `deps.go` compile without the old screen-stack.
